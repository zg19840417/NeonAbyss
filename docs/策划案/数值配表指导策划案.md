## 一、总览

### 1.1 Excel表格式规范（所有表通用）

| 行号 | 内容 |
|------|------|
| 第1行 | 字段名（英文，驼峰命名） |
| 第2行 | 数据类型：`int` / `float` / `string` / `bool` / `array` / `json` |
| 第3行 | 中文解释 |
| 第4行+ | 数据行 |

### 1.2 数据类型说明

| 类型 | 说明 | 示例 |
|------|------|------|
| `int` | 整数 | 100, 0, -1 |
| `float` | 浮点数（用小数表示百分比，如0.2=20%） | 0.2, 1.5 |
| `string` | 字符串 | "fire", "tank" |
| `bool` | 布尔值 | true, false |
| `array` | JSON数组 | `[{enemyId:"E001",level:1}]` |
| `json` | JSON对象 | `{"type":"mycelium","amount":100}` |

### 1.3 数据驱动流程

```
Excel (*.xlsx) → 转换脚本(tools/excelToJson.js) → JSON (*.json) → 代码引用
```

**禁止**：直接编辑 JSON 文件，所有修改必须在 Excel 中进行。

### 1.4 文件位置

- Excel源文件：`assets/data/excel/`
- JSON生成文件：`assets/data/json/`
- 转换命令：`npm run excel`（如可用）

## 三、各表详细字段规范

### 3.1 fusionGirls.xlsx — 融合姬基础配置

**说明**：每张融合姬卡的核心属性。计划100+张卡。

| 字段名 | 类型 | 说明 |
|--------|------|------|
| cardId | string | 唯一ID，如 "MC_fire_tank_001" |
| name | string | 融合姬名称（废土风格，全女性） |
| nameEn | string | 英文名 |
| quality | string | 品质：N / R / SR / SSR / UR / LE |
| element | string | 元素：water / fire / wind / light / dark |
| race | string | 种族：plant / animal / mech / energy / hybrid |
| subRace | string | 子种族：对应种族下的6个子种族之一 |
| profession | string | 职业：tank / dps / support |
| baseHp | int | 1级基础生命值 |
| baseAtk | int | 1级基础攻击力 |
| baseSpd | int | 1级基础速度 |
| hpGrowth | float | 每级生命成长系数 |
| atkGrowth | float | 每级攻击成长系数 |
| spdGrowth | float | 每级速度成长系数 |
| skillIds | array | 技能ID列表，如 ["SK001","SK002"] |

| description | string | 融合姬背景描述（废土风格，1-2句） |
| story | string | 详细剧情文本（可选，用于图鉴） |

**配表要点**：
- 品质越高，基础属性和成长系数越高
- 同品质下，不同职业的属性侧重不同（坦克高HP/DEF，输出高ATK，辅助均衡）
- 元素和种族的组合应合理分布，避免某种组合过多或缺失
- 每个品质至少覆盖5种元素 × 3种职业 = 15种组合

### 3.2 chips.xlsx — 强化芯片配置

**说明**：每张强化芯片的属性和技能。芯片效果作用于全队3个融合姬。

| 字段名 | 类型 | 说明 |
|--------|------|------|
| chipId | string | 唯一ID，如 "CHIP_SSR_ATK_01" |
| name | string | 芯片名称（废土风格） |
| nameEn | string | 英文名 |
| quality | string | 品质：N / R / SR / SSR / UR / LE |
| hpPercent | float | 生命百分比加成，如 0.15 = 15% |
| atkPercent | float | 攻击百分比加成 |
| targetProfession | string | 职业限定：null=通用 / tank / dps / support |
| targetElement | string | 元素限定：null=通用 / water / fire / wind / light / dark |
| targetRace | string | 种族限定：null=通用 / plant / animal / mech / energy / hybrid |
| skillIds | array | 芯片技能ID列表，引用chipSkills表 |
| description | string | 芯片描述 |

**配表要点**：
- 品质越高，属性百分比越高，技能数量越多（N/R/SR=1个，SSR/UR=2个，LE=3个）
- 有职业/元素/种族限定的芯片，属性百分比应比同品质通用芯片略高
- 芯片名称必须使用废土科幻风格，**禁止**：世界树、圣光、神之、永恒、创世等奇幻词汇
- 替换方向：神圣→净化/源初，永恒→不灭/不朽，创世→源初/起源

### 3.3 chipSkills.xlsx — 芯片技能配置

**说明**：芯片附带的技能效果。

| 字段名 | 类型 | 说明 |
|--------|------|------|
| skillId | string | 唯一ID，如 "CSK001" |
| name | string | 技能名称 |
| nameEn | string | 英文名 |
| type | string | 技能类型：aura（光环被动）/ trigger（触发型被动）/ modify（改变现有技能） |
| description | string | 技能效果描述（精确到数值） |
| triggerCondition | string/null | 触发条件（仅trigger类型）：onAttack / onDamageTaken / onKill / onTurnStart / onHpBelow / onBattleStart |
| effectType | string | 效果类型：buffAtk / buffDef / buffHp / buffSpd / buffCrit / buffDodge / damageReduction / heal / shield / trueDamage / extraAction 等 |
| value | float | 效果数值（百分比用小数，如0.2=20%） |
| duration | int | 持续回合数（0=永久/即时） |
| maxStack | int | 最大叠加层数（0=不可叠加） |
| cooldown | int | 冷却回合数（0=无冷却） |
| weight | int | 随机权重（用于事件池等场景） |

**配表要点**：
- 同样需要替换奇幻风格名称
- 描述必须精确到数值，如"提升佩戴者15%攻击力"而非"大幅提升攻击力"

### 3.4 enemies.xlsx — 敌人配置

**说明**：所有非Boss敌人。分为变异生物(mutant)和失心者(lost)两类。

| 字段名 | 类型 | 说明 |
|--------|------|------|
| enemyId | string | 唯一ID，如 "E001" |
| name | string | 敌人名称（废土风格） |
| nameEn | string | 英文名 |
| type | string | **必须**为：mutant（变异生物）或 lost（失心者） |
| element | string | 元素：water / fire / wind / light / dark |
| baseHp | int | 基础生命值 |
| baseAtk | int | 基础攻击力 |
| baseSpd | int | 基础速度 |
| skills | array | 技能ID列表（引用skills表），可为空 |
| description | string | 敌人背景描述（废土风格，说明变异/失心原因） |
| dropTable | array | 掉落表，如 [{type:"mycelium",min:10,max:30}] |

**配表要点**：
- type字段**只允许** mutant 和 lost 两个值，禁止使用旧的 mech/mutant/spirit/beast/undead
- 变异生物(mutant)：低级敌人，出现在野外，数值较低，无复杂技能
- 失心者(lost)：精英级敌人，出现在野外后期和禁区，数值较高，有技能
- 每张野外地图需要配置多种敌人，用于填充10个关卡（每关3个敌人）
- 敌人名称使用废土风格，**禁止**：亡灵、幽灵、灵魂、魔法等奇幻词汇

### 3.9 wildStages.xlsx — 野外关卡表（新建）

**说明**：10张野外地图 × 10个关卡 = 100条记录。

| 字段名 | 类型 | 说明 |
|--------|------|------|
| stageId | string | 唯一ID，如 "WS_01_05"（地图1第5关） |
| mapId | int | 地图编号（1-10） |
| mapName | string | 地图名称 |
| stageIndex | int | 关卡序号（1-10） |
| requiredLevel | int | 准入声望等级 |
| requiredStageId | string/null | 前置关卡ID（第1关为null） |
| enemies | array | 敌人配置，固定3个：[{enemyId, level}] |
| firstClearRewards | array | 首通奖励：[{type, id, amount}] |
| normalRewards | array | 通关奖励：[{type, id, amount}]（经验+货币） |
| unlockZoneId | string/null | 通关后解锁的禁区ID（第5和第10关有值） |

**准入等级规则**：
- 地图1第1关：1级，地图2第1关：11级，...地图10第1关：91级
- 同一地图内所有关卡准入等级相同

**敌人配置规则**：
- 每关固定3个敌人
- 前期关卡敌人等级低、数量少，后期关卡敌人等级高
- 敌人等级应与准入等级匹配，略高于准入等级

**奖励规则**：
- normalRewards：每关固定给经验和菌丝
- firstClearRewards：首次通关额外给源核等稀有奖励
- 第5关和第10关首通奖励应更丰厚

## 💻 程序实现指南

> 本节提取本文档中与程序代码实现相关的所有内容，以可直接执行的方式呈现。

### 一、Excel → JSON 转换系统

**转换流程**：
```
Excel (*.xlsx) → tools/excelToJson.js → JSON (*.json) → 代码引用
```

**文件位置**：
- Excel源文件目录：`assets/data/excel/`
- JSON生成文件目录：`assets/data/json/`
- 转换命令：`npm run excel`

**转换规则**：
1. 第1行读取为字段名（key），使用驼峰命名
2. 第2行读取为数据类型，用于JSON值的类型转换
3. 第3行读取为中文注释，可写入JSON的 `_comment` 字段或忽略
4. 第4行起为数据行，每行生成一个JSON对象
5. 生成的JSON文件名与Excel文件名一致（如 `fusionGirls.xlsx` → `fusionGirls.json`）
6. JSON结构为数组：`[{字段1: 值1, 字段2: 值2, ...}, ...]`

**数据类型转换规则**：

| Excel第2行类型 | JSON输出类型 | 转换逻辑 |
|---------------|-------------|---------|
| `int` | `number` | `parseInt(value)` |
| `float` | `number` | `parseFloat(value)` |
| `string` | `string` | 直接取值 |
| `bool` | `boolean` | `value === "true"` |
| `array` | `object` (数组) | `JSON.parse(value)` |
| `json` | `object` | `JSON.parse(value)` |

**禁止**：直接编辑 JSON 文件，所有数据修改必须在 Excel 中进行后重新转换。

### 二、数据结构规范

**字段命名规范**：
- 使用英文驼峰命名（camelCase）
- ID字段统一使用 `{前缀}_{描述}_{序号}` 格式
- 布尔字段使用 `is`/`has`/`can` 前缀（如 `isBoss`, `hasSkill`）
- 百分比字段使用 `Percent` 后缀（如 `hpPercent`, `atkPercent`）
- 外键引用字段使用关联表的ID字段名（如 `skillIds`, `cardId`, `poolId`）

**ID命名规范**：

| 表 | ID格式 | 示例 |
|----|--------|------|
| fusionGirls | `FG_{element}_{序号}` | `FG_water_001` |
| chips | `CHIP_{品质}_{属性}_{序号}` | `CHIP_SSR_ATK_01` |
| chipSkills | `CSK{序号}` | `CSK001` |
| enemies | `E{序号}` | `E001` |
| bosses | `BOSS_{序号}` | `BOSS_001` |
| skills | `SK{序号}` | `SK001` |
| items | `ITEM_{描述}` | `ITEM_RAD_MEDICINE` |
| wildStages | `WS_{地图}_{关卡}` | `WS_01_05` |
| zoneStages | `ZS_{禁区简称}_{层号}` | `ZS_LAB_01` |
| events | `EVT_{类型}_{序号}` | `EVT_BATTLE_001` |
| eventPools | `POOL_{禁区简称}` | `POOL_ZONE_LAB` |
| gachaPools | `POOL_{类型}` | `POOL_NORMAL` |
| zones | `ZONE_{简称}` | `ZONE_LAB` |

**加载机制**：
