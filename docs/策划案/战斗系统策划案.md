# 废土元年 — 战斗系统策划案

> **版本**：v2.0
> **版本日期**：2026-04-06
> **用途**：定义战斗系统的完整规则和框架

---

## 一、战斗概述

### 1.1 核心定位

- **类型**：纯自动回合制战斗，玩家不参与操作
- **队伍配置**：我方3张融合姬 + 1张强化芯片 vs 敌方配置
- **战斗时长**：120秒倒计时
- **速度控制**：可切换1x/2x速度
- **跳过**：不可跳过，必须观看战斗过程

### 1.2 胜负条件

| 条件 | 结果 |
|------|------|
| 消灭敌方全部单位 | 胜利 |
| 120秒倒计时结束，敌方仍有存活动单位 | 失败 |
| 我方3个融合姬全部倒下 | 失败 |

### 1.3 战斗无惩罚

- 失败无任何惩罚，可无限重试
- 不消耗体力/道具（关卡入场道具在进入时已消耗）
- 已通过的关卡不可重复挑战（野外），禁区可重置后重新挑战

## 二、基础属性

### 2.1 角色三维属性

每个融合姬和敌人只有**3个基础数值属性**：

| 属性 | 说明 |
|------|------|
| 生命（HP） | 角色的生命值，归零时角色倒下 |
| 攻击（ATK） | 角色的攻击力，影响所有伤害输出 |
| 速度（SPD） | 角色的行动速度，决定回合排序 |

**注意**：没有"防御力"属性。所有减伤、闪避、格挡等效果通过"能力"（被动）获得。

### 2.2 属性成长

- 融合姬通过升级提升三维属性
- 每个角色的属性成长率不同（由数值策划在配置表中设定）
- 强化芯片提供全队百分比加成（生命% + 攻击%）
- 品质越高，基础属性和成长率越高

### 2.3 其他战斗数值（通过能力获得）

以下数值**不是基础属性**，而是通过能力（被动）获得的：

| 数值 | 说明 | 典型来源 |
|------|------|---------|
| 百分比减伤 | 减少受到的伤害百分比 | 坦克能力 |
| 固定减伤 | 固定减少受到的伤害值 | 坦克能力 |
| 闪避率 | 百分比概率完全闪避攻击 | 敏捷型能力 |
| 命中率 | 百分比概率命中（对抗闪避） | 精准型能力 |
| 暴击率 | 百分比概率造成暴击 | 输出能力 |
| 暴击伤害 | 暴击时的伤害倍率（默认150%） | 输出能力 |
| 回复率 | 每回合回复生命百分比 | 辅助能力 |
| 吸血率 | 造成伤害时回复伤害值百分比 | 特殊能力 |

## 三、回合机制

### 3.1 速度排序

- 所有参战角色（我方+敌方）按**速度值从高到低**排序
- 速度最高的角色第一个行动，依次类推
- 每个回合所有角色行动一次后，重新按速度排序进入下一回合
- 速度相同的角色，随机决定先后顺序

### 3.2 行动流程

每个角色行动时的流程：

```
1. AI判定 → 选择技能
2. AI判定 → 选择目标
3. 执行技能 → 播放动画
4. 计算伤害 → 应用效果
5. 检查胜负条件
```

### 3.3 AI技能选择逻辑

AI按以下优先级选择技能：

1. **冷却完毕的技能**中，按优先级从高到低选择
2. 如果所有技能都在冷却中，使用普通攻击（1回合冷却，必定可用）
3. **辅助型技能**优先于攻击型技能（如果队友生命低于50%）
4. **大招/高冷却技能**在有利的时机使用（如敌方血量低于30%时）

### 3.4 AI目标选择逻辑

| 技能类型 | 目标选择 |
|---------|---------|
| 单体攻击 | 优先攻击生命百分比最低的敌人 |
| 群体攻击 | 自动攻击敌方全体 |
| 单体治疗 | 优先治疗生命百分比最低的队友 |
| 群体治疗 | 自动治疗我方全体 |
| 增益buff | 优先给攻击力最高的队友 |
| 减益debuff | 优先给攻击力最高的敌人 |

## 四、技能系统

### 4.1 技能分类

每个融合姬拥有两类能力：

| 类型 | 说明 | 触发方式 |
|------|------|---------|
| 技能 | 主动技能，战斗中自动施放 | AI控制，冷却完毕后自动使用 |
| 能力 | 被动效果，始终生效 | 自动生效，不需要触发 |

### 4.2 技能配置

每个融合姬固定拥有**3个技能**：

| 技能位 | 类型 | 冷却 | 说明 |
|--------|------|------|------|
| 技能1 | 普通攻击 | 1回合 | 基础攻击技能，伤害倍率较低，无特殊效果 |
| 技能2 | 主动技能 | 3-5回合 | 中等伤害/效果，可能有附加效果（debuff、回血等） |
| 技能3 | 主动技能 | 5-8回合 | 高伤害/强力效果，可能是大招级技能 |

**技能释放规则：**
- 每回合AI从冷却完毕的技能中随机选择一个施放
- 普通攻击（技能1）冷却为1回合，基本每回合都可用
- 技能2和技能3有较长冷却，不是每回合都能使用
- 技能可以附带触发效果（如攻击后30%概率附加流血）

### 4.3 能力（被动）配置

能力槽数量按品质区分：

| 品质 | 能力槽上限 | 说明 |
|------|-----------|------|
| N | 2个 | 基础被动 |
| R | 3个 | 1个额外被动 |
| SR | 4个 | 2个额外被动 |
| SSR | 4个 | 2个额外被动 |
| UR | 5个 | 3个额外被动 |
| LE | 6个 | 4个额外被动 |

**能力来源：**
- **天生能力**：角色生成时随机抽取的被动能力，品质决定能力槽数量
- **合成获得**：异化合成（SSR级）可能获得额外能力（70%无额外/25%被动/5%主动）

**能力类型示例：**

| 类型 | 示例 | 效果 |
|------|------|------|
| 生存型 | 坚韧 | 永久减少受到伤害10% |
| 生存型 | 闪避本能 | 15%概率闪避单体攻击 |
| 输出型 | 暴击强化 | 暴击率+15% |
| 输出型 | 穿透 | 攻击无视目标20%减伤 |
| 辅助型 | 治愈光环 | 每回合为生命最低的队友回复攻击力10%的生命 |
| 辅助型 | 速度鼓舞 | 开战时为我方全体提升10%速度，持续3回合 |
| 特殊型 | 反伤 | 受到攻击时反弹受到伤害的20% |
| 特殊型 | 吸血 | 造成伤害的15%转化为自身生命 |

## 五、伤害计算

### 5.1 基础伤害公式

```
最终伤害 = 攻击力 × 技能倍率 × 克制系数 × 暴击系数 × 增减益系数
```

### 5.2 各系数说明

#### 技能倍率

由数值策划在技能配置表中设定，不同技能不同倍率。

| 技能类型 | 典型倍率范围 |
|---------|-------------|
| 普通攻击 | 0.8 - 1.2 |
| 主动技能2 | 1.5 - 2.5 |
| 主动技能3（大招） | 2.5 - 4.0 |
| 群体攻击 | 单体的60%-80% per target |

#### 克制系数

| 关系 | 系数 |
|------|------|
| 克制对方 | ×1.2 |
| 被对方克制 | ×1.0（不额外减伤，只是对方打你时×1.2） |
| 无克制关系 | ×1.0 |
| 光暗互克 | 攻击克制方×1.2 |

#### 暴击系数

| 情况 | 系数 |
|------|------|
| 未暴击 | ×1.0 |
| 暴击 | ×暴击伤害（默认150%，可通过能力提升） |

#### 增减益系数

| 效果 | 系数 |
|------|------|
| 攻击力提升buff | ×(1 + buff百分比) |
| 敌方减伤debuff | ×(1 - debuff百分比) |
| 百分比减伤（能力） | ×(1 - 减伤百分比) |
| 固定减伤（能力） | 伤害 - 固定值（最低为1） |

### 5.3 伤害计算示例

```
我方融合姬：攻击力1000，火元素
敌方敌人：无减伤能力
技能：主动技能3，倍率3.0
克制关系：火克风 → 克制系数1.2
暴击：暴击率20%，触发暴击，暴击伤害150%

最终伤害 = 1000 × 3.0 × 1.2 × 1.5 = 5400
```

## 六、强化芯片在战斗中的作用

### 6.1 基础加成

强化芯片为全队3个融合姬提供**全额**加成：

| 属性 | 加成方式 |
|------|---------|
| 生命 | 芯片生命% × 融合姬基础生命 |
| 攻击 | 芯片攻击% × 融合姬基础攻击 |

### 6.2 芯片技能

芯片拥有全局被动技能，作用于全队：

| 品质 | 技能数量 | 说明 |
|------|---------|------|
| N | 1个 | 基础光环 |
| R | 1个 | 基础光环 |
| SR | 1个 | 中级光环 |
| SSR | 2个 | 中级光环+触发型被动 |
| UR | 2个 | 高级光环+触发型被动 |
| LE | 3个 | 高级光环+触发型被动+改变技能 |

**芯片技能类型：**
- **光环被动**：始终生效的全队buff（如全队攻击+10%）
- **触发型被动**：满足条件时触发（如队友生命低于30%时，为其施加护盾）
- **改变现有技能**：修改融合姬技能的效果（如将某融合姬的火属性技能变为火+光双属性）

## 七、元素克制

### 7.1 克制关系

```
  水 ←克→ 火 ←克→ 风
  ↑                   ↓
  克                 克
  ↑                   ↓
  暗 ←互克→ 光
```

### 7.2 克制效果

- 攻击克制方：最终伤害 ×1.2
- 被克制方攻击你：对方伤害 ×1.2（即你受到更多伤害）
- 无克制关系：伤害 ×1.0
- 克制效果在所有计算完成后应用

### 7.3 克制在战斗中的体现

- UI上显示克制关系图标（攻击时闪烁克制提示）
- 克制攻击有额外的视觉特效（元素色光圈）
- 组队时考虑元素克制是核心策略之一

## 八、战斗流程示意

```
┌─────────────────────────────────────────┐
│              战斗开始                      │
│  1. 显示双方阵容                           │
│  2. 按速度排序生成行动条                    │
│  3. 应用开场buff（芯片光环等）              │
├─────────────────────────────────────────┤
│              回合循环                      │
│  1. 当前角色行动                           │
│     → AI选择技能（优先冷却完毕的高优先级）    │
│     → AI选择目标（按技能类型规则）           │
│     → 执行技能 → 播放动画                  │
│     → 计算伤害 → 扣血/治疗/buff/debuff     │
│     → 检查角色倒下 → 移除行动条             │
│  2. 检查胜负条件                           │
│     → 敌方全灭 → 胜利                      │
│     → 我方全灭 → 失败                      │
│  3. 下一个角色行动                         │
│  4. 所有角色行动完毕 → 新回合               │
│  5. 检查120秒倒计时                        │
│     → 超时 → 失败                          │
├─────────────────────────────────────────┤
│              战斗结束                      │
│  胜利 → 展示结算面板（经验/货币/掉落）       │
│  失败 → 展示失败面板（可重试）              │
└─────────────────────────────────────────┘
```

## 九、与配置表的关联

### 9.1 相关配置表

| 配置表 | 关键字段 | 说明 |
|--------|---------|------|
| minionCards.xlsx | hp, atk, spd, skill1Id, skill2Id, skill3Id, abilityIds | 融合姬战斗属性 |
| skills.xlsx | skillId, name, type, cooldown, multiplier, target, effect, triggerChance | 技能配置 |
| chips.xlsx | hpPercent, atkPercent, skillIds | 芯片战斗属性 |
| chipSkills.xlsx | skillId, name, type, condition, effect | 芯片技能配置 |
| enemies.xlsx | hp, atk, spd, skillIds | 敌人战斗属性 |
| bosses.xlsx | hp, atk, spd, skillIds, phaseSkills | Boss战斗属性（含阶段技能） |

### 9.2 数值策划配表要点

1. **技能倍率**需要平衡各品质、各职业之间的差距
2. **能力效果**的数值需要控制在不破坏平衡的范围内
3. **Boss技能**需要分阶段设计（血量低于50%/25%时切换技能模式）
4. **速度值**决定行动顺序，需要合理分配避免某些角色永远先手
5. **芯片加成**的百分比需要控制在不使低品质角色超越高品质角色的范围内

### 四、元素克制系数（配入 elementRelations.xlsx）

| 关系 | 系数 |
|------|------|

### 五、芯片加成公式（配入 chips.xlsx）

芯片为全队3个融合姬提供全额加成：
- 最终HP = 融合姬 baseHp x (1 + 芯片 hpPercent)
- 最终ATK = 融合姬 baseAtk x (1 + 芯片 atkPercent)

芯片技能数量按品质：N/R=1个，SR=1个，SSR/UR=2个，LE=3个

### 六、伤害公式完整版（程序实现参考）

```
最终伤害 = 攻击力 x 技能倍率 x 克制系数 x 暴击系数 x 增减益系数
```

其中：
- 攻击力 = 融合姬 baseAtk x (1 + 芯片 atkPercent) x (1 + buff百分比)
- 克制系数：克制=1.2，无克制=1.0
- 暴击系数：未暴击=1.0，暴击=暴击伤害值（默认150%）
- 增减益系数 = (1 - 敌方减伤百分比) x (1 - 敌方debuff百分比)
- 固定减伤：伤害 - 固定值（最低为1）

### 七、战斗表现与交互定稿

本节用于给策划确认战斗界面的最终信息结构，若与旧版描述冲突，以本节为准。

#### 1. 顶层信息结构

- 顶部固定显示：`关卡名`、`当前回合数`
- 中部战斗演示区固定为：`敌方3张大卡` + `中间演示区` + `我方3张大卡`
- 战斗演示区下方固定显示：`单行战斗日志`
- 底部常驻按钮固定为：`1x/2x速度切换`、`暂停`

#### 2. 日志与暂停的统一规则

- 点击`单行战斗日志`，等同于点击`暂停`
- 两者统一触发以下行为：
  - 战斗暂停
  - 日志面板向上展开，覆盖战斗演示区
  - 底部按钮切换为：`继续`、`投降`

```javascript
function selectSkill(unit) {
    // 1. 获取所有可用技能（冷却完毕的）
    const availableSkills = unit.skills.filter(s => s.cooldownRemaining === 0);
    
    if (availableSkills.length === 0) {
        // 2. 所有技能都在冷却中，使用普攻（冷却1回合，必定可用）
        return unit.skills[0]; // 技能1为普攻
    }
    
    // 3. 检查是否有队友生命低于50%
    const allyHpLow = unit.allies.some(ally => ally.hpPercent < 0.5);
    
    if (allyHpLow) {
        // 优先使用辅助型技能（治疗/buff）
        const supportSkill = availableSkills.find(s => s.effectType === 'heal' || s.effectType === 'buff');
        if (supportSkill) return supportSkill;
    }
    
    // 4. 检查是否有敌方血量低于30%
    const enemyHpLow = unit.enemies.some(enemy => enemy.hpPercent < 0.3);
    
    if (enemyHpLow) {
        // 优先使用大招/高冷却技能
        const ultimateSkill = availableSkills
            .filter(s => s.skillIndex === 2) // 技能3为大招
            .sort((a, b) => b.cooldown - a.cooldown)[0];
        if (ultimateSkill) return ultimateSkill;
    }
    
    // 5. 默认：按技能优先级从高到低选择（技能3 > 技能2 > 技能1）
    return availableSkills.sort((a, b) => b.priority - a.priority)[0];
}
```

### 三、AI目标选择逻辑

```javascript
function selectTarget(skill, unit) {
    switch (skill.targetType) {
        case 'singleEnemy':
            // 优先攻击生命百分比最低的敌人
            return unit.enemies
                .filter(e => e.isAlive)
                .sort((a, b) => a.hpPercent - b.hpPercent)[0];
        
        case 'allEnemies':
            // 攻击敌方全体
            return unit.enemies.filter(e => e.isAlive);
        
        case 'self':
            return unit;
        
        case 'singleAlly':
            // 优先治疗生命百分比最低的队友
            return unit.allies
                .filter(a => a.isAlive)
                .sort((a, b) => a.hpPercent - b.hpPercent)[0];
        
        case 'allAllies':
            return unit.allies.filter(a => a.isAlive);
        
        // buff/debuff 特殊处理
        default:
            if (skill.effectType === 'buff') {
                // 增益：优先给攻击力最高的队友
                return unit.allies
                    .filter(a => a.isAlive)
                    .sort((a, b) => b.atk - a.atk)[0];
            }
            if (skill.effectType === 'debuff') {
                // 减益：优先给攻击力最高的敌人
                return unit.enemies
                    .filter(e => e.isAlive)
                    .sort((a, b) => b.atk - a.atk)[0];
            }
    }
}
```

### 四、伤害计算函数

```javascript
function calculateDamage(attacker, defender, skill) {
    let damage = attacker.atk * skill.baseMultiplier;
    
    // 克制系数
    const elementRelation = getElementRelation(attacker.element, defender.element);
    if (elementRelation === 'advantage') {
        damage *= 1.2;
    }
    
    // 暴击判定
    let isCrit = false;
    if (Math.random() < attacker.critRate) {
        isCrit = true;
        damage *= attacker.critDamage; // 默认1.5
    }
    
    // 攻击力buff
    if (attacker.atkBuff) {
        damage *= (1 + attacker.atkBuff);
    }
    
    // 敌方百分比减伤
    if (defender.damageReduction) {
        damage *= (1 - defender.damageReduction);
    }
    
    // 敌方固定减伤
    if (defender.fixedReduction) {
        damage -= defender.fixedReduction;
    }
    
    // 最低伤害为1
    damage = Math.max(1, Math.floor(damage));
    
    return { damage, isCrit, isAdvantage: elementRelation === 'advantage' };
}
```

### 七、Boss战特殊逻辑

```javascript
// Boss阶段切换（仅BOSS_010有3阶段）
checkBossPhase(boss) {
    if (boss.bossId !== 'BOSS_010') return;
    
    const hpPercent = boss.hp / boss.maxHp;
    
    if (hpPercent <= 0.25 && boss.currentPhase < 3) {
        boss.currentPhase = 3;
        this.triggerBossTransform(boss, 3); // 播放变身动画
        boss.skills = this.getPhaseSkills(boss.bossId, 3); // 切换技能组
    } else if (hpPercent <= 0.5 && boss.currentPhase < 2) {
        boss.currentPhase = 2;
        this.triggerBossTransform(boss, 2);
        boss.skills = this.getPhaseSkills(boss.bossId, 2);
    }
}
```

### 八、战斗无惩罚规则

- 失败不消耗任何资源（关卡入场道具在进入时已消耗）
- 可无限重试
- 野外已通关关卡不可重复挑战
- 禁区可重置后重新挑战
