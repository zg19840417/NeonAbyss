> **版本**：v2.0
> **版本日期**：2026-04-06
> **用途**：指导数值策划为每种职业的融合姬配置能力（被动），AI可直接据此配表

---

## 一、能力系统概述

### 1.1 什么是能力

能力是融合姬的**被动效果**，始终在战斗中生效，不需要手动触发。每个融合姬根据品质拥有不同数量的能力槽。

### 1.2 能力槽数量

| 品质 | 能力槽 | 说明 |
|------|--------|------|
| N | 2 | 1个职业通用能力 + 1个种族/元素能力 |
| R | 3 | 1个职业通用能力 + 1个种族/元素能力 + 1个随机能力 |
| SR | 4 | 1个职业通用能力 + 1个种族/元素能力 + 2个随机能力 |
| SSR | 4 | 1个职业核心能力 + 1个种族/元素能力 + 2个随机能力 |
| UR | 5 | 1个职业核心能力 + 1个种族/元素能力 + 3个随机能力 |
| LE | 6 | 1个职业终极能力 + 1个种族/元素能力 + 4个随机能力 |

### 1.3 能力来源分类

| 来源 | 说明 | 占槽 |
|------|------|------|
| 职业能力 | 根据职业从对应能力树中选取 | 必占1个槽 |
| 种族/元素能力 | 根据种族或元素特性获得 | 必占1个槽 |
| 随机能力 | 从通用能力池中随机选取 | 占剩余槽 |
| 后天学习 | 通过升级/任务/合成获得 | 替换随机能力 |
| 异化额外 | 异化SSR可能获得的额外能力 | 不占槽（额外） |

### 1.4 能力数据结构（配置表字段）

```
abilityId: string       — 能力唯一ID（如 "TANK_DEF_01"）
name: string            — 能力名称
description: string     — 能力描述（中文）
type: string            — 能力类型（buff/debuff/trigger/heal/special）
target: string          — 作用目标（self/ally_all/ally_lowest_hp/enemy_all）
condition: string       — 触发条件（always/on_hit/on_crit/on_ally_hp_below_30%/on_kill 等）
effect: string          — 效果类型（damage_reduce_percent/damage_reduce_fixed/dodge/crit_rate/crit_damage/heal_percent/lifesteal/reflect 等）
value: float            — 效果数值（如 0.1 = 10%）
duration: int           — 持续回合数（0=永久）
stackable: bool         — 是否可叠加
maxStack: int           — 最大叠加层数
source: string          — 来源（class/race/element/random/learned/mutation）
```

## 二、坦克（Tank）能力树

坦克的核心职责是**承受伤害、保护队友**。坦克不追求高输出，而是通过减伤、嘲讽、护盾等手段吸引敌方攻击并存活。

## 六、配表指南（AI数值策划专用）

### 6.1 配置流程

为每个融合姬配置能力时，按以下步骤：

```
步骤1：确定品质 → 查看能力槽数量
步骤2：根据职业 → 从职业能力树中选取1个职业通用/核心/终极能力
步骤3：根据种族 → 从种族能力池中选取1个种族能力
步骤4：根据元素 → 从元素能力池中选取1个元素能力（与种族能力二选一，占同一个槽）
步骤5：剩余槽 → 从通用能力池或职业方向能力树中随机选取
步骤6：检查 → 确保能力之间不冲突、不重复
```

### 6.2 能力选取优先级建议

| 品质 | 职业 | 种族/元素 | 随机1 | 随机2 | 随机3 | 随机4 |
|------|------|----------|-------|-------|-------|-------|
| N | 通用能力 | 种族或元素 | — | — | — | — |
| R | 通用能力 | 种族或元素 | 方向能力 | — | — | — |
| SR | 通用能力 | 种族或元素 | 方向能力 | 方向能力 | — | — |
| SSR | 核心能力 | 种族或元素 | 方向能力 | 方向能力 | — | — |
| UR | 核心能力 | 种族或元素 | 方向能力 | 方向能力 | 方向能力 | — |
| LE | 终极能力 | 种族或元素 | 方向能力 | 方向能力 | 方向能力 | 方向能力 |

### 6.3 能力冲突检查

以下能力**不可同时配置**在同一角色上：

| 冲突组 | 原因 |
|--------|------|
| TANK_DEF_01 + TANK_DEF_02 | 同类型减伤，取最高不叠加 |
| DPS_CRIT_01 + DPS_CRIT_02 | 同类型暴击率，取最高不叠加 |
| SUP_HEAL_01 + SUP_HEAL_02 | 同类型治疗，取最高不叠加 |
| RACE_PLANT_01 + RACE_MECH_01 | 不同种族能力不可混搭 |
| ELEM_WATER_01 + ELEM_FIRE_01 | 不同元素能力不可混搭 |

### 6.4 配置表字段映射

能力在 `minionCards.xlsx` 中以 `abilityIds` 字段存储，格式为JSON数组：

```json

// SSR品质输出示例

// LE品质辅助示例
{"abilityIds": ["SUP_ULT_01", "RACE_ANIMAL_01", "SUP_A_02", "SUP_B_03", "SUP_C_01", "DPS_SPD_01"]}
```
