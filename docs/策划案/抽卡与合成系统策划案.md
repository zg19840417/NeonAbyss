## 一、系统概述

### 1.1 获取融合姬的两种方式

| 方式 | 说明 | 产出品质 |
|------|------|---------|
| 源核抽卡 | 消耗源核在卡池中抽取 | R / SR / SSR / UR |
| 合成系统 | 消耗同品质卡牌+菌丝（详见《主副卡融合系统策划案》） | N→R→SR→SSR→UR→LE |

### 1.2 融合姬强化方式

融合姬获取后，可通过以下方式进一步强化：

| 方式 | 说明 | 详见 |
|------|------|------|
| 改造系统（打书） | 使用改造晶片为卡牌学习新能力 | 本文档 三、改造系统（打书） |
| 异化 | 通过改造系统学习隐藏能力实现异化 | 本文档 三、改造系统（打书） |

### 1.3 N卡获取方式

N卡**不在卡池中出现**，通过以下方式获取：
- 野外关卡首次通关奖励
- 禁区事件奖励
- 每日任务奖励
- 成就奖励
- 新手引导赠送

## 二、抽卡系统

### 2.1 卡池类型

#### 常驻卡池

- **名称**：融合召唤
- **开放时间**：始终可用
- **内容**：包含所有非LE品质的融合姬（R/SR/SSR/UR）
- **货币**：源核
- **价格**：单抽200源核，十连2000源核
- **限定角色**：限定UP结束后，该角色自动加入常驻池

#### 限定UP卡池

- **名称**：按UP角色命名（如"锈叶·苏蔓 限定召唤"）
- **开放时间**：限时开放（通常2周）
- **内容**：常驻池全部内容 + UP角色概率提升
- **货币**：源核
- **价格**：单抽200源核，十连2000源核
- **UP规则**：
  - UP角色为SR品质时：该SR角色出现概率从基础概率提升至UP概率
  - UP角色为SSR品质时：该SSR角色出现概率从基础概率提升至UP概率
  - UP角色为UR品质时：该UR角色出现概率从基础概率提升至UP概率
- **UP结束后**：UP角色自动加入常驻池，原概率恢复

### 2.2 基础概率（常驻池）

| 品质 | 单抽概率 | 说明 |
|------|---------|------|
| R | 85% | 基础品质 |
| SR | 12% | 精良品质 |
| SSR | 2.5% | 史诗品质 |
| UR | 0.5% | 传说品质 |
| **合计** | **100%** | |

### 2.3 保底机制

| 保底类型 | 触发条件 | 效果 |
|---------|---------|------|
| SR保底 | 每10抽未出SR，第10抽必出SR | 硬保底，计数器独立 |
| SSR保底 | 每40抽未出SSR，第40抽必出SSR | 硬保底，计数器独立 |
| UR保底 | 每200抽未出UR，第200抽必出UR | 硬保底，计数器独立 |

**保底规则说明：**
- 三种保底计数器**互相独立**，互不影响
- 出现对应品质或更高品质时，计数器**重置为0**
- 例如：第8抽出了SSR，则SR计数器和SSR计数器都重置，UR计数器继续累计
- 保底在**常驻池和限定池之间共享计数器**（切换卡池不重置）
- 新服/新玩家首次十连**必出1张SR**

### 2.4 限定UP概率（示例）

以SSR角色UP为例：

| 品质 | 常驻概率 | UP池概率 | 说明 |
|------|---------|---------|------|
| R | 85% | 85% | 不变 |
| SR | 12% | 12% | 不变 |
| SSR（非UP） | 2.5% | 0.5% | 分摊给UP角色 |
| SSR（UP角色） | — | 2.0% | 从SSR总概率中分出 |
| UR | 0.5% | 0.5% | 不变 |

**注意**：UP角色的额外概率从同品质总概率中分出，不增加总概率。

### 2.5 抽卡动画与演出

| 品质 | 抽卡演出 |
|------|---------|
| R | 白光闪烁，卡牌快速翻转 |
| SR | 蓝色光柱，卡牌缓慢翻转，有粒子特效 |
| SSR | 金色光柱+粒子风暴+屏幕震动，卡牌特写展示 |
| UR | 彩虹光柱+全屏特效+专属BGM片段，卡牌全屏展示 |

### 2.6 重复卡处理

抽到已拥有的融合姬时：
- **重复卡转化为"碎片"**，碎片按品质区分（R碎片/SR碎片/SSR碎片/UR碎片）
- 碎片用途：
  - 在"碎片商店"中兑换指定角色的碎片（集齐N个碎片可解锁对应角色）
  - 兑换其他资源（菌丝、源核等）

| 品质 | 重复卡→碎片数量 | 解锁所需碎片 |
|------|---------------|-------------|
| R | 1个R碎片 | 20个R碎片 |
| SR | 1个SR碎片 | 40个SR碎片 |
| SSR | 1个SSR碎片 | 80个SSR碎片 |
| UR | 1个UR碎片 | 150个UR碎片 |

## 三、合成系统

### 3.1 融合姬合成

> **融合姬合成已迁移至《主副卡融合系统策划案》，详见该文档。**

### 3.2 芯片合成

**公式**：3张同品质任意芯片 → 1张高一品质随机芯片（不消耗菌丝和改造晶片）

| 合成方向 | 材料芯片 | 菌丝消耗 | 产出 |
|---------|---------|---------|------|
| N → R | 3张N芯片 | 0 | 1张随机R芯片 |
| R → SR | 3张R芯片 | 0 | 1张随机SR芯片 |
| SR → SSR | 3张SR芯片 | 0 | 1张随机SSR芯片 |
| SSR → UR | 3张SSR芯片 | 0 | 1张随机UR芯片 |
| UR → LE | 3张UR芯片 | 0 | 1张随机LE芯片 |

**规则说明：**
- 3张材料芯片**必须为同一品质**
- 产出为**随机**高一品质的芯片
- 合成后材料芯片**消失**，不可恢复
- 芯片合成**不消耗菌丝和改造晶片**，固定需要3张同品质材料

### 3.3 改造系统（打书）

> **设计参考**：类似《梦幻西游》魔兽要诀打书系统。改造晶片 = 能力书，使用后为卡牌学习对应能力。

#### 3.3.1 核心概念

改造晶片是独立于融合系统的能力强化道具。每个改造晶片对应游戏中的一个能力（技能），使用改造晶片可以为卡牌"打书"——即让卡牌学会该能力。

**核心规则：**
- **一对一映射**：一个改造晶片 = 一个能力，每种能力有唯一对应的晶片
- **无品质限制**：改造晶片可用于任意品质的卡牌（N/R/SR/SSR/UR/LE 均可使用）
- **消耗品**：使用后改造晶片消失，不可回收
- **不可重复学习**：同一能力不能被同一卡牌学习两次

#### 3.3.2 能力槽与覆盖机制

每张卡牌拥有固定数量的能力槽，分为以下两类：

| 槽位类型 | 说明 | 数量规则 |
|---------|------|---------|
| 必带能力槽 (forcedAbilities) | 卡牌固有能力，不可被覆盖 | 由卡牌配置表决定（通常1-2个） |
| 可扩展能力槽 (learnedAbilities) | 通过打书学习的能力，可被覆盖 | 卡牌总能力上限 - 必带能力数量 |

**打书结果判定：**

| 卡牌状态 | 打书结果 | 风险 |
|---------|---------|------|
| 有空槽位 | 新能力填入空槽位，卡牌获得新能力 | **无风险** |
| 槽位已满 | 新能力**随机替换**一个已学能力（非必带能力） | **有覆盖风险** |

**覆盖概率表**（当卡牌槽位已满时触发）：

| 覆盖目标 | 覆盖概率 | 说明 |
|---------|---------|------|
| 替换一个 普通能力（R级） | 50% | 普通能力最容易被覆盖 |
| 替换一个 核心能力（SR级） | 30% | 核心能力较难被覆盖 |
| 替换一个 隐藏能力（SSR级） | 0% | 隐藏能力不会被覆盖（但可能失败） |
| 打书失败（卡牌不变，晶片消耗） | 20% | 完全失败，晶片白白消耗 |

**覆盖规则详细说明：**

1. **必带能力（forcedAbilities）永远不会被覆盖**，这是铁律
2. 当卡牌槽位已满时，系统按上述概率表随机决定结果
3. 如果被覆盖的目标类型中存在多个能力，则从中**随机选择一个**进行替换
4. **隐藏能力（SSR级）不可被覆盖**，但打书SSR晶片时如果卡牌已有SSR能力，则直接判定为失败（20%概率）
5. **新晶片品质越高，失败概率越低**（见下方修正表）

**品质修正表**（对失败概率的修正）：

| 新晶片品质 | 失败概率修正 | 实际失败概率 |
|-----------|------------|------------|
| R晶片 | 基础值 | 20% |
| SR晶片 | -5% | 15% |
| SSR晶片 | -10% | 10% |

> **注意**：品质修正仅降低失败概率，不改变各类型能力的覆盖概率比例。覆盖概率按 50:30:0 的比例重新归一化分配。

修正后的覆盖概率示例（以SR晶片为例，失败率15%）：

| 覆盖目标 | 覆盖概率（归一化后） |
|---------|-------------------|
| 替换 普通能力 | 50% × 85% = 42.5% |
| 替换 核心能力 | 30% × 85% = 25.5% |
| 打书失败 | 15% |
| **合计** | **82% + 15% = 但需归一化** |

实际计算方式：
- 有效覆盖总权重 = 50 + 30 = 80
- 普通能力覆盖概率 = (50/80) × (1 - 失败率)
- 核心能力覆盖概率 = (30/80) × (1 - 失败率)
- 失败概率 = 基础失败率 - 品质修正

#### 3.3.3 改造晶片品质与能力类型

改造晶片的品质决定对应能力的类型和稀有度：

| 晶片品质 | 对应能力类型 | 覆盖风险等级 | 说明 |
|---------|------------|------------|------|
| R晶片 | 普通能力 | 低风险 | 基础被动能力，如生命加成、攻击加成、防御加成等 |
| SR晶片 | 核心能力 | 中风险 | 强力职业专属能力，如职业联动技能、元素共鸣等 |
| SSR晶片 | 隐藏能力 | 高风险 | 超稀有融合能力，学习后卡牌获得异化特征 |

**各品质能力举例：**

| 晶片品质 | 能力ID示例 | 能力名称 | 能力效果 |
|---------|-----------|---------|---------|
| R | ABILITY_HP_UP_01 | 生命强化·壹 | 生命值 +5% |
| R | ABILITY_ATK_UP_01 | 攻击强化·壹 | 攻击力 +5% |
| R | ABILITY_DEF_UP_01 | 防御强化·壹 | 防御力 +5% |
| SR | ABILITY_TANK_SHIELD | 坦克护盾 | 受到伤害时，有30%概率获得最大生命10%的护盾 |
| SR | ABILITY_DPS_CRIT | 暴击本能 | 暴击率 +15%，暴击伤害 +20% |
| SR | ABILITY_SUPPORT_HEAL | 治愈光环 | 回合开始时，为生命最低的友方恢复攻击力20%的生命 |
| SSR | ABILITY_MUTATE_BERSERK | 异化·狂暴 | 攻击力 +30%，但受到伤害 +15%，外观变为异化形态 |
| SSR | ABILITY_MUTATE_PHANTOM | 异化·幻影 | 每回合有20%概率闪避攻击，闪避后反击50%伤害 |

#### 3.3.4 异化机制

异化不再是独立的角色系统，而是通过改造打书系统实现的：

- 当卡牌通过打书学习了**SSR级隐藏能力**时，卡牌获得"异化"标记
- 异化标记的视觉效果：
  - 卡牌边框变为异化专属样式（暗金+红色纹路）
  - 立绘添加异化特征（变异光芒、暗红/紫色光效）
  - 能力图标添加异化标识
- 异化标记**可逆**：如果该SSR能力被后续打书覆盖，异化标记消失
- 异化**不影响卡牌基础属性**（不再有120%属性加成），仅提供SSR能力本身的数值效果

#### 3.3.5 改造晶片获取途径

| 晶片类型 | 获取途径 | 产出概率/数量 | 说明 |
|---------|---------|-------------|------|
| R晶片（普通能力） | 野外关卡掉落 | 每关约15%概率 | 最基础的获取方式 |
| R晶片（普通能力） | 每日任务奖励 | 每日任务随机产出1-2个 | 稳定的日常来源 |
| R晶片（普通能力） | 商店购买 | 500菌丝/个 | 菌丝消耗出口 |
| SR晶片（核心能力） | 禁区Boss掉落 | Boss首次击杀必掉1个 | 每个Boss对应不同职业的SR晶片 |
| SR晶片（核心能力） | 禁区事件奖励 | 完成事件概率产出 | 约30%概率 |
| SR晶片（核心能力） | 活动商店兑换 | 活动代币兑换 | 限时活动期间可用 |
| SSR晶片（隐藏能力） | 高级禁区掉落（地图6+） | Boss概率掉落，约5% | 最稀有的获取方式 |
| SSR晶片（隐藏能力） | 限时活动 | 活动排名/积分奖励 | 特殊活动独占 |
| SSR晶片（隐藏能力） | 碎片合成 | 3个SSR晶片碎片 → 1个SSR晶片 | 碎片通过高级关卡产出 |

#### 3.3.6 打书操作流程

```
玩家进入改造界面
    │
    ├─ 选择要改造的卡牌
    │     └─ 显示卡牌当前所有能力（必带 + 已学）
    │
    ├─ 选择要使用的改造晶片
    │     └─ 显示晶片对应的能力信息
    │
    ├─ 系统预览（previewOverride）
    │     ├─ 如果有空槽位 → 显示"将填入空槽位"（绿色提示）
    │     └─ 如果槽位已满 → 显示覆盖概率预览：
    │           ├─ "替换 普通能力 的概率：XX%"（黄色提示）
    │           ├─ "替换 核心能力 的概率：XX%"（橙色提示）
    │           └─ "打书失败 的概率：XX%"（红色提示）
    │
    ├─ 玩家确认使用
    │     └─ 消耗改造晶片
    │
    └─ 播放打书动画 + 显示结果
          ├─ 成功学习 → 新能力图标飞入卡牌
          ├─ 覆盖成功 → 旧能力图标碎裂 + 新能力图标飞入
          └─ 打书失败 → 晶片碎裂动画 + 卡牌不变
```

#### 3.3.7 打书确认界面信息展示

确认界面需展示以下信息，确保玩家知情：

| 展示项 | 说明 |
|--------|------|
| 卡牌当前能力列表 | 包含必带能力（锁定标识）和已学能力 |
| 即将使用的能力 | 晶片对应的能力名称、品质、效果描述 |
| 结果预览 | 有空槽位时显示"安全学习"；满槽时显示覆盖概率 |
| 覆盖目标高亮 | 满槽时，可能被覆盖的能力高亮显示 |
| 失败警告 | 满槽时，用红色文字提示"有XX%概率打书失败，晶片消耗" |

#### 3.3.8 特殊规则与限制

1. **N卡打书**：N卡也可以打书，但N卡能力槽上限较低（通常只有1-2个可扩展槽位）
2. **打书顺序无限制**：不限制先打R再打SR，玩家可自由选择打书顺序
3. **能力唯一性**：同一能力ID不能重复学习，如果卡牌已有该能力，使用对应晶片会提示"已学会该能力"并拒绝操作
4. **晶片不可回收**：已使用的晶片无法通过任何方式找回
5. **打书不影响等级和突破**：打书与卡牌等级、突破进度完全独立
6. **异化能力数量上限**：每张卡牌最多只能拥有1个SSR级隐藏能力（避免无限叠加异化效果）

---

> **注意**：异化角色系统已整合至改造打书系统中。通过使用SSR级改造晶片学习隐藏能力，卡牌即可获得异化特征。详见 3.3.4 异化机制。

## 四、碎片商店

### 4.1 商店内容

玩家可以用重复卡转化来的碎片，在碎片商店中兑换指定角色。

| 兑换项 | 价格 | 说明 |
|--------|------|------|
| 指定R角色 | 20个R碎片 | 可选择任意已有R角色 |
| 指定SR角色 | 40个SR碎片 | 可选择任意已有SR角色 |
| 指定SSR角色 | 80个SSR碎片 | 可选择任意已有SSR角色 |
| 指定UR角色 | 150个UR碎片 | 可选择任意已有UR角色 |

> **注意**：改造晶片已从碎片商店移除，改为通过关卡掉落、任务奖励、活动获取等途径获得（详见 3.3.5 改造晶片获取途径）。

### 4.2 商店刷新

- 碎片商店**不刷新**，所有角色始终可兑换
- 新获得的融合姬会自动加入碎片商店的可兑换列表

### 三、保底计数器规则（3个独立计数器）

| 计数器 | 触发条件 | 保底效果 | 重置条件 |
|--------|---------|---------|---------|
| SR计数器 | 每10抽未出SR | 第10抽必出SR | 出现SR或更高品质时重置为0 |
| SSR计数器 | 每40抽未出SSR | 第40抽必出SSR | 出现SSR或更高品质时重置为0 |
| UR计数器 | 每200抽未出UR | 第200抽必出UR | 出现UR时重置为0 |

**规则细节**：
- 三个计数器互相独立，互不影响
- 出现对应品质或更高品质时，该计数器及以下所有计数器重置
- 例：第8抽出了SSR → SR计数器重置 + SSR计数器重置，UR计数器继续累计
- 常驻池和限定池之间共享计数器（切换卡池不重置）
- 新服/新玩家首次十连必出1张SR

### 七、打书覆盖概率配置（配入 chipBooks.xlsx 或独立配置表）

#### 基础覆盖概率

| 配置项 | 值 | 说明 |
|--------|-----|------|
| overrideRate.common | 50% | 替换普通能力（R级）的概率权重 |
| overrideRate.core | 30% | 替换核心能力（SR级）的概率权重 |
| overrideRate.hidden | 0% | 替换隐藏能力（SSR级）的概率权重（不可覆盖） |
| failRate.base | 20% | 基础失败概率 |

#### 品质修正系数

| 新晶片品质 | 失败概率修正 | 实际失败概率 |
|-----------|------------|------------|
| R | 0% | 20% |
| SR | -5% | 15% |
| SSR | -10% | 10% |

#### 覆盖概率计算公式

```
实际失败概率 = max(0, failRate.base - qualityFailBonus[chipQuality])
有效覆盖总权重 = overrideRate.common + overrideRate.core + overrideRate.hidden
普通能力覆盖概率 = (overrideRate.common / 有效覆盖总权重) × (1 - 实际失败概率)
核心能力覆盖概率 = (overrideRate.core / 有效覆盖总权重) × (1 - 实际失败概率)
```

```javascript
    // 单抽
    draw(poolId) {
        const pool = ConfigManager.getPoolById(poolId);
        const items = ConfigManager.getGachaItemsByPool(poolId);
        
        // 1. 确定品质
        const quality = this.rollQuality(pool);
        
        // 2. 在该品质的所有卡中随机选择
        const candidates = items.filter(i => i.quality === quality);
        const card = this.weightedRandom(candidates);

        this.updatePityCounters(quality);
        
        // 4. 检查是否重复
        if (this.isDuplicate(card.cardId)) {
            return this.convertToFragment(card);
        }
        
        return card;
    }
    
    // 十连抽
    drawTen(poolId) {
        const results = [];
        for (let i = 0; i < 10; i++) {
            results.push(this.draw(poolId));
        }
        // 新玩家首次十连保底：如果没有SR及以上，强制替换最后一张为SR
        if (this.isFirstTenDraw && !results.some(r => r.quality === 'SR' || ...)) {
            results[9] = this.forceDraw(poolId, 'SR');
        }
        return results;
    }
}
```

### 二、保底计数器逻辑（3个独立计数器）

```javascript
class PitySystem {
    constructor() {
        this.counters = {
            SR: { current: 0, limit: 10, softPityStart: 7 },
            SSR: { current: 0, limit: 40, softPityStart: 28 },
            UR: { current: 0, limit: 200, softPityStart: 140 }
        };
    }
    
    // 抽卡后更新计数器
    update(quality) {
        // 品质等级映射
        const qualityRank = { R: 1, SR: 2, SSR: 3, UR: 4 };
        const drawnRank = qualityRank[quality];
        
        // 遍历所有计数器
        for (const [key, counter] of Object.entries(this.counters)) {
            const pityRank = qualityRank[key];
            
            if (drawnRank >= pityRank) {
                // 出现该品质或更高品质，重置计数器
                counter.current = 0;
            } else {
                // 未出现，计数器+1
                counter.current += 1;
            }
        }
    }
    
    // 软保底概率递增
    getSoftPityBonus(quality) {
        const counter = this.counters[quality];
        if (counter.current >= counter.softPityStart) {
            // 从软保底起始开始，每抽增加概率
            const progress = (counter.current - counter.softPityStart) / (counter.limit - counter.softPityStart);
            return progress * 0.5; // 最大额外50%概率
        }
        return 0;
    }
    
    // 检查是否触发硬保底
    isHardPity(quality) {
        return this.counters[quality].current >= this.counters[quality].limit;
    }
}
```

**关键实现要点**：
- 三个计数器（SR/SSR/UR）互相独立，存储在玩家存档中
- 常驻池和限定池共享同一套计数器
- 切换卡池时不清零计数器
- 新玩家标记 `isFirstTenDraw` 在首次十连后置为 false

```javascript
function rollQualityWithUP(pool) {
    const baseRates = { R: 0.85, SR: 0.12, SSR: 0.025, UR: 0.005 };
    let rates = { ...baseRates };
    
    if (pool.poolType === 'limited' && pool.upCharId) {
        const upCard = ConfigManager.getCardById(pool.upCharId);
        const upQuality = upCard.quality;
        
        // UP角色从同品质总概率中分出
        const upRate = pool.upRate; // 如SSR UP时为0.02
        const nonUpRate = baseRates[upQuality] - upRate; // 剩余概率给非UP角色
        
        rates[upQuality] = upRate; // UP角色概率
        rates[`nonUp_${upQuality}`] = nonUpRate; // 非UP角色概率
    }
    
    // 加上软保底加成
    const pityBonus = pitySystem.getSoftPityBonus(currentCheckingQuality);
    rates[currentCheckingQuality] += pityBonus;
    
    // 按概率随机选择品质
    return weightedRandom(rates);
}
```

### 六、改造系统逻辑（ChipBookSystem）

```javascript
class ChipBookSystem {
    constructor() {
        // 覆盖概率配置
        this.overrideConfig = {
            rates: {
                common: 50,   // 替换普通能力（R级）的权重
                core: 30,     // 替换核心能力（SR级）的权重
                hidden: 0     // 替换隐藏能力（SSR级）的权重（不可覆盖）
            },
            failRate: {
                base: 0.20,   // 基础失败概率 20%
                qualityBonus: {
                    R: 0,      // R晶片无修正
                    SR: -0.05, // SR晶片失败率 -5%
                    SSR: -0.10 // SSR晶片失败率 -10%
                }
            }
        };
    }

    /**
     * 使用改造晶片（打书）
     * @param {string} cardId - 卡牌实例ID
     * @param {string} chipBookId - 改造晶片ID
     * @returns {Object} 打书结果 { success, result, replacedAbility, newAbility }
     */
    useChip(cardId, chipBookId) {
        const card = inventory.getCard(cardId);
        const chipBook = ConfigManager.getChipBook(chipBookId);

        // 1. 前置检查
        if (!card) throw new Error('卡牌不存在');
        if (!chipBook) throw new Error('改造晶片不存在');
        if (!inventory.hasChipBook(chipBookId)) throw new Error('未持有该改造晶片');

        // 2. 检查是否已学会该能力
        if (card.learnedAbilities.includes(chipBook.abilityId) ||
            card.forcedAbilities.includes(chipBook.abilityId)) {
            throw new Error('该卡牌已学会此能力');
        }

        // 3. 检查SSR异化能力上限（每张卡最多1个SSR隐藏能力）
        if (chipBook.quality === 'SSR' && chipBook.isMutated) {
            const hasMutatedAbility = card.learnedAbilities.some(abilityId => {
                const ability = ConfigManager.getAbility(abilityId);
                return ability && ability.category === 'hidden';
            });
            if (hasMutatedAbility) {
                throw new Error('该卡牌已有隐藏能力，无法学习新的隐藏能力');
            }
        }

        // 4. 消耗改造晶片
        inventory.removeChipBook(chipBookId, 1);

        // 5. 判断是否有空槽位
        const maxSlots = card.maxAbilitySlots;
        const usedSlots = card.forcedAbilities.length + card.learnedAbilities.length;
        const hasEmptySlot = usedSlots < maxSlots;

        if (hasEmptySlot) {
            // 有空槽位：直接学习，无风险
            card.learnedAbilities.push(chipBook.abilityId);
            this.updateMutatedStatus(card);
            return {
                success: true,
                result: 'learned',
                newAbility: chipBook.abilityId,
                card: card
            };
        }

        // 6. 槽位已满：计算覆盖概率
        const overrideResult = this.calculateOverride(card, chipBook);

        if (overrideResult.result === 'fail') {
            // 打书失败
            return {
                success: false,
                result: 'failed',
                card: card
            };
        }

        // 7. 执行覆盖
        const targetAbilityId = overrideResult.targetAbilityId;
        const targetIndex = card.learnedAbilities.indexOf(targetAbilityId);
        card.learnedAbilities[targetIndex] = chipBook.abilityId;
        this.updateMutatedStatus(card);

        return {
            success: true,
            result: 'overridden',
            replacedAbility: targetAbilityId,
            newAbility: chipBook.abilityId,
            card: card
        };
    }

    /**
     * 计算覆盖结果（槽位已满时调用）
     * @param {Object} card - 卡牌数据
     * @param {Object} chipBook - 改造晶片配置
     * @returns {Object} { result: 'override'|'fail', targetAbilityId, probabilities }
     */
    calculateOverride(card, chipBook) {
        const config = this.overrideConfig;

        // 计算实际失败概率
        const failBonus = config.failRate.qualityBonus[chipBook.quality] || 0;
        const actualFailRate = Math.max(0, config.failRate.base + failBonus);

        // 分类已学能力（排除必带能力）
        const learnedAbilities = card.learnedAbilities.map(id => ({
            abilityId: id,
            ability: ConfigManager.getAbility(id)
        }));

        const commonAbilities = learnedAbilities.filter(a => a.ability.category === 'common');
        const coreAbilities = learnedAbilities.filter(a => a.ability.category === 'core');
        const hiddenAbilities = learnedAbilities.filter(a => a.ability.category === 'hidden');

        // 计算有效覆盖总权重
        const totalWeight = config.rates.common + config.rates.core + config.rates.hidden;

        // 如果没有可覆盖的能力（全部是hidden），直接失败
        if (totalWeight === 0) {
            return { result: 'fail', probabilities: { fail: 1.0 } };
        }

        // 计算各类型覆盖概率
        const commonRate = (config.rates.common / totalWeight) * (1 - actualFailRate);
        const coreRate = (config.rates.core / totalWeight) * (1 - actualFailRate);

        // 构建概率分布
        const roll = Math.random();

        // 判定是否失败
        if (roll < actualFailRate) {
            return {
                result: 'fail',
                probabilities: {
                    common: commonRate,
                    core: coreRate,
                    fail: actualFailRate
                }
            };
        }

        // 在覆盖成功的情况下，选择覆盖目标
        const effectiveRoll = (roll - actualFailRate) / (1 - actualFailRate);
        let targetAbilityId = null;

        if (effectiveRoll < config.rates.common / totalWeight && commonAbilities.length > 0) {
            // 覆盖普通能力
            const target = commonAbilities[Math.floor(Math.random() * commonAbilities.length)];
            targetAbilityId = target.abilityId;
        } else if (coreAbilities.length > 0) {
            // 覆盖核心能力
            const target = coreAbilities[Math.floor(Math.random() * coreAbilities.length)];
            targetAbilityId = target.abilityId;
        } else if (commonAbilities.length > 0) {
            // 没有核心能力可覆盖，回退到普通能力
            const target = commonAbilities[Math.floor(Math.random() * commonAbilities.length)];
            targetAbilityId = target.abilityId;
        } else {
            // 没有可覆盖的能力
            return { result: 'fail', probabilities: { fail: 1.0 } };
        }

        return {
            result: 'override',
            targetAbilityId: targetAbilityId,
            probabilities: {
                common: commonRate,
                core: coreRate,
                fail: actualFailRate
            }
        };
    }

    /**
     * 预览覆盖概率（不实际执行）
     * @param {string} cardId - 卡牌实例ID
     * @param {string} chipBookId - 改造晶片ID
     * @returns {Object} 预览结果 { hasEmptySlot, probabilities, possibleTargets }
     */
    previewOverride(cardId, chipBookId) {
        const card = inventory.getCard(cardId);
        const chipBook = ConfigManager.getChipBook(chipBookId);

        if (!card || !chipBook) return null;

        // 检查是否已学会
        const alreadyLearned = card.learnedAbilities.includes(chipBook.abilityId) ||
                               card.forcedAbilities.includes(chipBook.abilityId);
        if (alreadyLearned) {
            return { error: 'already_learned' };
        }

        // 检查SSR上限
        if (chipBook.quality === 'SSR' && chipBook.isMutated) {
            const hasMutated = card.learnedAbilities.some(id => {
                const a = ConfigManager.getAbility(id);
                return a && a.category === 'hidden';
            });
            if (hasMutated) {
                return { error: 'mutated_limit_reached' };
            }
        }

        const maxSlots = card.maxAbilitySlots;
        const usedSlots = card.forcedAbilities.length + card.learnedAbilities.length;
        const hasEmptySlot = usedSlots < maxSlots;

        if (hasEmptySlot) {
            return {
                hasEmptySlot: true,
                emptySlots: maxSlots - usedSlots,
                result: 'safe_learn'
            };
        }

        // 槽位已满，计算覆盖概率
        const config = this.overrideConfig;
        const failBonus = config.failRate.qualityBonus[chipBook.quality] || 0;
        const actualFailRate = Math.max(0, config.failRate.base + failBonus);
        const totalWeight = config.rates.common + config.rates.core + config.rates.hidden;

        const commonAbilities = card.learnedAbilities.filter(id => {
            const a = ConfigManager.getAbility(id);
            return a && a.category === 'common';
        });
        const coreAbilities = card.learnedAbilities.filter(id => {
            const a = ConfigManager.getAbility(id);
            return a && a.category === 'core';
        });

        return {
            hasEmptySlot: false,
            result: 'override_risk',
            probabilities: {
                common: totalWeight > 0 ? (config.rates.common / totalWeight) * (1 - actualFailRate) : 0,
                core: totalWeight > 0 ? (config.rates.core / totalWeight) * (1 - actualFailRate) : 0,
                fail: actualFailRate
            },
            possibleTargets: {
                common: commonAbilities.map(id => ConfigManager.getAbility(id)),
                core: coreAbilities.map(id => ConfigManager.getAbility(id))
            }
        };
    }

    /**
     * 更新卡牌的异化状态
     * @param {Object} card - 卡牌数据
     */
    updateMutatedStatus(card) {
        const hasMutatedAbility = card.learnedAbilities.some(abilityId => {
            const ability = ConfigManager.getAbility(abilityId);
            return ability && ability.category === 'hidden';
        });
        card.isMutated = hasMutatedAbility;

        // 如果异化状态变化，触发视觉更新
        if (card.isMutated !== card._prevMutatedStatus) {
            EventSystem.emit('mutated_status_changed', {
                cardId: card.instanceId,
                isMutated: card.isMutated
            });
            card._prevMutatedStatus = card.isMutated;
        }
    }
}
```
