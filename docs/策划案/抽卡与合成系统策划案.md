# 废土元年 — 抽卡与合成系统策划案

> **版本**：v2.0
> **版本日期**：2026-04-06
> **用途**：定义抽卡招募和合成系统的完整规则

---

## 一、系统概述

### 1.1 获取融合姬的两种方式

| 方式 | 说明 | 产出品质 |
|------|------|---------|
| 源核抽卡 | 消耗源核在卡池中抽取 | R / SR / SSR / UR |
| 合成系统 | 消耗同品质卡牌+菌丝+可选改造晶片 | N→R→SR→SSR→UR→LE |

### 1.2 N卡获取方式

N卡**不在卡池中出现**，通过以下方式获取：
- 野外关卡首次通关奖励
- 禁区事件奖励
- 每日任务奖励
- 成就奖励
- 新手引导赠送

## 二、抽卡系统

### 2.1 卡池类型

#### 常驻卡池

- **名称**：融合召唤
- **开放时间**：始终可用
- **内容**：包含所有非LE品质的融合姬（R/SR/SSR/UR）
- **货币**：源核
- **价格**：单抽200源核，十连2000源核
- **限定角色**：限定UP结束后，该角色自动加入常驻池

#### 限定UP卡池

- **名称**：按UP角色命名（如"锈叶·苏蔓 限定召唤"）
- **开放时间**：限时开放（通常2周）
- **内容**：常驻池全部内容 + UP角色概率提升
- **货币**：源核
- **价格**：单抽200源核，十连2000源核
- **UP规则**：
  - UP角色为SR品质时：该SR角色出现概率从基础概率提升至UP概率
  - UP角色为SSR品质时：该SSR角色出现概率从基础概率提升至UP概率
  - UP角色为UR品质时：该UR角色出现概率从基础概率提升至UP概率
- **UP结束后**：UP角色自动加入常驻池，原概率恢复

### 2.2 基础概率（常驻池）

| 品质 | 单抽概率 | 说明 |
|------|---------|------|
| R | 85% | 基础品质 |
| SR | 12% | 精良品质 |
| SSR | 2.5% | 史诗品质 |
| UR | 0.5% | 传说品质 |
| **合计** | **100%** | |

### 2.3 保底机制

| 保底类型 | 触发条件 | 效果 |
|---------|---------|------|
| SR保底 | 每10抽未出SR，第10抽必出SR | 硬保底，计数器独立 |
| SSR保底 | 每40抽未出SSR，第40抽必出SSR | 硬保底，计数器独立 |
| UR保底 | 每200抽未出UR，第200抽必出UR | 硬保底，计数器独立 |

**保底规则说明：**
- 三种保底计数器**互相独立**，互不影响
- 出现对应品质或更高品质时，计数器**重置为0**
- 例如：第8抽出了SSR，则SR计数器和SSR计数器都重置，UR计数器继续累计
- 保底在**常驻池和限定池之间共享计数器**（切换卡池不重置）
- 新服/新玩家首次十连**必出1张SR**

### 2.4 限定UP概率（示例）

以SSR角色UP为例：

| 品质 | 常驻概率 | UP池概率 | 说明 |
|------|---------|---------|------|
| R | 85% | 85% | 不变 |
| SR | 12% | 12% | 不变 |
| SSR（非UP） | 2.5% | 0.5% | 分摊给UP角色 |
| SSR（UP角色） | — | 2.0% | 从SSR总概率中分出 |
| UR | 0.5% | 0.5% | 不变 |

**注意**：UP角色的额外概率从同品质总概率中分出，不增加总概率。

### 2.5 抽卡动画与演出

| 品质 | 抽卡演出 |
|------|---------|
| R | 白光闪烁，卡牌快速翻转 |
| SR | 蓝色光柱，卡牌缓慢翻转，有粒子特效 |
| SSR | 金色光柱+粒子风暴+屏幕震动，卡牌特写展示 |
| UR | 彩虹光柱+全屏特效+专属BGM片段，卡牌全屏展示 |

### 2.6 重复卡处理

抽到已拥有的融合姬时：
- **重复卡转化为"碎片"**，碎片按品质区分（R碎片/SR碎片/SSR碎片/UR碎片）
- 碎片用途：
  - 在"碎片商店"中兑换指定角色的碎片（集齐N个碎片可解锁对应角色）
  - 兑换其他资源（菌丝、源核等）

| 品质 | 重复卡→碎片数量 | 解锁所需碎片 |
|------|---------------|-------------|
| R | 1个R碎片 | 20个R碎片 |
| SR | 1个SR碎片 | 40个SR碎片 |
| SSR | 1个SSR碎片 | 80个SSR碎片 |
| UR | 1个UR碎片 | 150个UR碎片 |

## 三、合成系统

### 3.1 基础合成规则

**公式**：4张同品质卡 + 菌丝 → 1张高一品质的随机卡

注意：以上合成规则仅适用于融合姬卡。强化芯片的合成规则为：3张同品质任意芯片 → 1张高一品质随机芯片（不消耗菌丝和改造晶片）。

| 合成方向 | 材料卡 | 菌丝消耗 | 产出 |
|---------|--------|---------|------|
| N → R | 4张N卡 | 1000菌丝 | 1张随机R卡 |
| R → SR | 4张R卡 | 5000菌丝 | 1张随机SR卡 |
| SR → SSR | 4张SR卡 | 20000菌丝 | 1张随机SSR卡 |
| SSR → UR | 4张SSR卡 | 100000菌丝 | 1张随机UR卡 |
| UR → LE | 4张UR卡 | 500000菌丝 | 1张随机LE卡 |

**规则说明：**
- 4张材料卡**必须为同一品质**，但种族/元素/职业可以不同
- 产出为**随机**高一品质的融合姬（从该品质全部角色池中随机）
- 合成后材料卡**消失**，不可恢复
- 产出角色**可能重复**（已有角色转化为碎片）

### 3.2 定向合成（改造晶片系统）

在基础合成中加入**改造晶片**，可以定向影响合成结果。

#### 改造晶片分类

| 晶片类型 | 对应材料品质 | 效果 | 获取方式 |
|---------|-------------|------|---------|
| 职业改造晶片·R | R级合成 | 合成结果限定为**特定职业**（坦克/输出/辅助三选一） | 关卡掉落、任务奖励 |
| 种族改造晶片·SR | SR级合成 | 合成结果限定为**特定种族**（5种族选1） | 禁区Boss掉落、活动奖励 |
| 元素改造晶片·SR | SR级合成 | 合成结果限定为**特定元素**（5元素选1） | 禁区Boss掉落、活动奖励 |
| 异化改造晶片·SSR | SSR级合成 | 合成结果为**异化角色**（属性强化+可能额外技能） | 高级禁区掉落、限时活动 |

#### 定向合成公式

**R级定向合成：**
```
4张N卡 + 1000菌丝 + 职业改造晶片·R（坦克/输出/辅助）→ 1张指定职业的随机R卡
```

**SR级定向合成：**
```
4张R卡 + 5000菌丝 + 种族改造晶片·SR（指定种族）→ 1张指定种族的随机SR卡
4张R卡 + 5000菌丝 + 元素改造晶片·SR（指定元素）→ 1张指定元素的随机SR卡
```
（不可同时使用种族和元素晶片，二选一）

**SSR级异化合成：**
```
4张SR卡 + 20000菌丝 + 异化改造晶片·SSR → 1张随机异化SSR卡
```

#### 异化角色说明

异化角色是SSR品质的特殊变体，与普通SSR角色的区别：

| 属性 | 普通SSR | 异化SSR |
|------|---------|---------|
| 生命值 | 基础值 | 基础值 × 120% |
| 攻击力 | 基础值 | 基础值 × 120% |
| 技能数量 | 基础技能 | 基础技能 + 可能获得额外技能 |
| 外观 | 普通立绘 | 异化立绘（视觉上有明显变异特征） |
| 卡框 | SSR金色卡框 | 异化专属卡框（暗金+红色纹路） |

**异化额外技能概率：**
- 70%概率：无额外技能（仅属性提升）
- 25%概率：获得1个额外被动技能
- 5%概率：获得1个额外主动技能

### 3.3 改造晶片获取

| 晶片类型 | 获取方式 | 大致产出 |
|---------|---------|---------|
| 职业改造晶片·R | 野外关卡概率掉落 | 每关约5%概率 |
| 种族改造晶片·SR | 禁区事件奖励/Boss首次击杀 | 每个禁区Boss首次击杀必掉1个 |
| 元素改造晶片·SR | 禁区事件奖励/Boss首次击杀 | 每个禁区Boss首次击杀必掉1个 |
| 异化改造晶片·SSR | 高级禁区（地图6+）Boss概率掉落 | 约10%概率 |

### 三、保底计数器规则（3个独立计数器）

| 计数器 | 触发条件 | 保底效果 | 重置条件 |
|--------|---------|---------|---------|
| SR计数器 | 每10抽未出SR | 第10抽必出SR | 出现SR或更高品质时重置为0 |
| SSR计数器 | 每40抽未出SSR | 第40抽必出SSR | 出现SSR或更高品质时重置为0 |
| UR计数器 | 每200抽未出UR | 第200抽必出UR | 出现UR时重置为0 |

**规则细节**：
- 三个计数器互相独立，互不影响
- 出现对应品质或更高品质时，该计数器及以下所有计数器重置
- 例：第8抽出了SSR → SR计数器重置 + SSR计数器重置，UR计数器继续累计
- 常驻池和限定池之间共享计数器（切换卡池不重置）
- 新服/新玩家首次十连必出1张SR

## 🎨 美术执行指南

### 一、抽卡演出特效（按品质分级）

#### R品质演出
- **背景**：白色光芒闪烁
- **动画**：卡牌快速翻转（约0.5秒）
- **音效**：基础翻牌音效
- **粒子**：少量白色粒子飘散
- **整体时长**：约1-2秒

#### SR品质演出
- **背景**：蓝色光柱从底部升起
- **动画**：卡牌缓慢翻转（约1秒），有停顿感
- **音效**：品质提升音效（比R更华丽）
- **粒子**：蓝色粒子特效环绕卡牌
- **整体时长**：约2-3秒

#### SSR品质演出
- **背景**：金色光柱 + 屏幕震动
- **动画**：卡牌特写展示，从远处飞入屏幕中央
- **音效**：史诗级音效，有冲击感
- **粒子**：金色粒子风暴，全屏范围
- **屏幕效果**：屏幕轻微震动
- **整体时长**：约3-4秒

#### UR品质演出
- **背景**：彩虹光柱 + 全屏特效
- **动画**：卡牌全屏展示，多角度旋转特写
- **音效**：专属BGM片段（约5秒），传说级音效
- **粒子**：彩虹色粒子覆盖全屏
- **屏幕效果**：屏幕强烈震动 + 闪光
- **整体时长**：约4-5秒

#### 十连抽演出
- 10张卡依次翻开，从低品质到高品质排列
- 最后一张为最高品质时，播放对应品质的完整演出
- 中间过程可加速跳过（保留最后一张的完整演出）

### 一、抽卡完整流程

```javascript
class GachaSystem {
    // 单抽
    draw(poolId) {
        const pool = ConfigManager.getPoolById(poolId);
        const items = ConfigManager.getGachaItemsByPool(poolId);
        
        // 1. 确定品质
        const quality = this.rollQuality(pool);
        
        // 2. 在该品质的所有卡中随机选择
        const candidates = items.filter(i => i.quality === quality);
        const card = this.weightedRandom(candidates);
        
        // 3. 更新保底计数器
        this.updatePityCounters(quality);
        
        // 4. 检查是否重复
        if (this.isDuplicate(card.cardId)) {
            return this.convertToFragment(card);
        }
        
        return card;
    }
    
    // 十连抽
    drawTen(poolId) {
        const results = [];
        for (let i = 0; i < 10; i++) {
            results.push(this.draw(poolId));
        }
        // 新玩家首次十连保底：如果没有SR及以上，强制替换最后一张为SR
        if (this.isFirstTenDraw && !results.some(r => r.quality === 'SR' || ...)) {
            results[9] = this.forceDraw(poolId, 'SR');
        }
        return results;
    }
}
```

### 二、保底计数器逻辑（3个独立计数器）

```javascript
class PitySystem {
    constructor() {
        this.counters = {
            SR: { current: 0, limit: 10, softPityStart: 7 },
            SSR: { current: 0, limit: 40, softPityStart: 28 },
            UR: { current: 0, limit: 200, softPityStart: 140 }
        };
    }
    
    // 抽卡后更新计数器
    update(quality) {
        // 品质等级映射
        const qualityRank = { R: 1, SR: 2, SSR: 3, UR: 4 };
        const drawnRank = qualityRank[quality];
        
        // 遍历所有计数器
        for (const [key, counter] of Object.entries(this.counters)) {
            const pityRank = qualityRank[key];
            
            if (drawnRank >= pityRank) {
                // 出现该品质或更高品质，重置计数器
                counter.current = 0;
            } else {
                // 未出现，计数器+1
                counter.current += 1;
            }
        }
    }
    
    // 软保底概率递增
    getSoftPityBonus(quality) {
        const counter = this.counters[quality];
        if (counter.current >= counter.softPityStart) {
            // 从软保底起始开始，每抽增加概率
            const progress = (counter.current - counter.softPityStart) / (counter.limit - counter.softPityStart);
            return progress * 0.5; // 最大额外50%概率
        }
        return 0;
    }
    
    // 检查是否触发硬保底
    isHardPity(quality) {
        return this.counters[quality].current >= this.counters[quality].limit;
    }
}
```

**关键实现要点**：
- 三个计数器（SR/SSR/UR）互相独立，存储在玩家存档中
- 常驻池和限定池共享同一套计数器
- 切换卡池时不清零计数器
- 新玩家标记 `isFirstTenDraw` 在首次十连后置为 false

### 三、限定UP概率分摊算法

```javascript
function rollQualityWithUP(pool) {
    const baseRates = { R: 0.85, SR: 0.12, SSR: 0.025, UR: 0.005 };
    let rates = { ...baseRates };
    
    if (pool.poolType === 'limited' && pool.upCharId) {
        const upCard = ConfigManager.getCardById(pool.upCharId);
        const upQuality = upCard.quality;
        
        // UP角色从同品质总概率中分出
        const upRate = pool.upRate; // 如SSR UP时为0.02
        const nonUpRate = baseRates[upQuality] - upRate; // 剩余概率给非UP角色
        
        rates[upQuality] = upRate; // UP角色概率
        rates[`nonUp_${upQuality}`] = nonUpRate; // 非UP角色概率
    }
    
    // 加上软保底加成
    const pityBonus = pitySystem.getSoftPityBonus(currentCheckingQuality);
    rates[currentCheckingQuality] += pityBonus;
    
    // 按概率随机选择品质
    return weightedRandom(rates);
}
```

### 四、合成逻辑

```javascript
class SynthesisSystem {
    // 融合姬卡合成
    synthesizeCards(materialCardIds, chipItemId = null) {
        // 1. 验证材料
        const materials = materialCardIds.map(id => inventory.getCard(id));
        
        // 检查是否4张同品质
        const qualities = materials.map(m => m.quality);
        if (!qualities.every(q => q === qualities[0])) {
            throw new Error('材料品质必须相同');
        }
        if (materials.length !== 4) {
            throw new Error('需要4张材料卡');
        }
        
        // 2. 检查菌丝是否足够
        const quality = qualities[0];
        const cost = SYNTHESIS_COST[quality]; // {N:1000, R:5000, SR:20000, SSR:100000, UR:500000}
        if (inventory.getMycelium() < cost) {
            throw new Error('菌丝不足');
        }
        
        // 3. 确定合成方向
        const targetQuality = UPGRADE_MAP[quality]; // {N:'R', R:'SR', SR:'SSR', SSR:'UR', UR:'LE'}
        let pool = getAllCardsByQuality(targetQuality);
        
        // 4. 应用改造晶片效果
        if (chipItemId) {
            const chip = ConfigManager.getItemById(chipItemId);
            pool = this.applyChipFilter(pool, chip);
        }
        
        // 5. 随机产出
        const result = weightedRandom(pool);
        
        // 6. 消耗材料和菌丝
        materialCardIds.forEach(id => inventory.removeCard(id));
        inventory.removeMycelium(cost);
        if (chipItemId) inventory.removeItem(chipItemId);
        
        // 7. 检查重复
        if (inventory.hasCard(result.cardId)) {
            return this.convertToFragment(result);
        }
        
        return result;
    }
    
    // 芯片合成（3合1，不消耗菌丝）
    synthesizeChips(materialChipIds) {
        const materials = materialChipIds.map(id => inventory.getChip(id));
        const qualities = materials.map(m => m.quality);
        
        if (!qualities.every(q => q === qualities[0]) || materials.length !== 3) {
            throw new Error('需要3张同品质芯片');
        }
        
        const targetQuality = UPGRADE_MAP[qualities[0]];
        const pool = getAllChipsByQuality(targetQuality);
        const result = weightedRandom(pool);
        
        materialChipIds.forEach(id => inventory.removeChip(id));
        return result;
    }
    
    // 改造晶片过滤
    applyChipFilter(pool, chip) {
        switch (chip.subType) {
            case 'professionPlugin':
                return pool.filter(c => c.profession === chip.targetProfession);
            case 'racePlugin':
                return pool.filter(c => c.race === chip.targetRace);
            case 'elementPlugin':
                return pool.filter(c => c.element === chip.targetElement);
            case 'mutatePlugin':
                // 异化合成：产出异化角色
                return pool.filter(c => c.isMutated);
            default:
                return pool;
        }
    }
}
```

### 五、碎片转化逻辑

```javascript
class FragmentSystem {
    // 重复卡→碎片
    convertToFragment(card) {
        const fragmentType = `${card.quality}Fragment`;
        const fragmentAmount = 1; // 每张重复卡固定1个碎片
        
        inventory.addFragment(fragmentType, fragmentAmount);
        return { type: 'fragment', fragmentType, amount: fragmentAmount, sourceCard: card };
    }
    
    // 碎片→角色
    exchangeFragment(cardId) {
        const card = ConfigManager.getCardById(cardId);
        const requiredFragments = FRAGMENT_COST[card.quality]; // {R:20, SR:40, SSR:80, UR:150}
        const fragmentType = `${card.quality}Fragment`;
        
        if (inventory.getFragment(fragmentType) < requiredFragments) {
            throw new Error('碎片不足');
        }
        
        inventory.removeFragment(fragmentType, requiredFragments);
        inventory.addCard(cardId);
    }
}
```

function generateMutatedCard(baseCard) {
    const mutatedCard = { ...baseCard };
    
    // 属性提升120%
    mutatedCard.baseHp = Math.floor(baseCard.baseHp * 1.2);
    mutatedCard.baseAtk = Math.floor(baseCard.baseAtk * 1.2);
    mutatedCard.baseSpd = Math.floor(baseCard.baseSpd * 1.2);
    
    // 标记为异化
    mutatedCard.isMutated = true;
    mutatedCard.mutatedPortrait = `${baseCard.portrait}_mutated`;
    
    // 额外技能概率判定
    const roll = Math.random();
    if (roll < 0.05) {
        // 5%概率：获得1个额外主动技能
        mutatedCard.extraSkillIds = [getRandomActiveSkill()];
    } else if (roll < 0.30) {
        // 25%概率：获得1个额外被动技能
        mutatedCard.extraSkillIds = [getRandomPassiveSkill()];
    }
    // 70%概率：无额外技能（仅属性提升）
    
    return mutatedCard;
}
