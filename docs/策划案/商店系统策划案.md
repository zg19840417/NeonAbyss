> **版本日期**：2026-04-06
> **用途**：定义商店系统的完整规则和商品配置

---

## 一、系统概述

### 1.1 商店入口

商店是庇护所主界面的功能入口之一，位于底部导航栏。

### 1.2 商店类型

| 商店 | 使用货币 | 刷新规则 | 说明 |
|------|---------|---------|------|
| 源核商店 | 源核 | 每日限量 | 购买高级道具 |
| 菌丝商店 | 菌丝 | 每日限量 | 购买基础物资 |
| 星币商店 | 星币 | 每日限量 | 购买付费礼包 |
| 碎片商店 | 碎片 | 不刷新 | 兑换指定角色（详见抽卡策划案） |

### 1.3 刷新规则

- **刷新时间**：每天凌晨5:00（服务器时间）
- **限量说明**：每个商品有每日购买次数上限，刷新后重置
- **刷新提示**：刷新前1小时显示倒计时提示

## 五、碎片商店

### 5.1 概述

碎片商店用于将重复卡转化来的碎片兑换为指定角色。

### 5.2 兑换规则

| 兑换项 | 价格 | 说明 |
|--------|------|------|
| 指定R角色 | 20个R碎片 | 可选择任意已有R角色 |
| 指定SR角色 | 40个SR碎片 | 可选择任意已有SR角色 |
| 指定SSR角色 | 80个SSR碎片 | 可选择任意已有SSR角色 |
| 指定UR角色 | 150个UR碎片 | 可选择任意已有UR角色 |
| 职业改造晶片·R（随机） | 10个R碎片 | 随机职业 |
| 菌丝 | 1个R碎片 → 500菌丝 | 基础兑换 |
| 源核 | 5个SR碎片 → 100源核 | 高级兑换 |

### 5.3 刷新规则

- 碎片商店**不刷新**
- 所有已解锁角色始终可兑换
- 新获得的融合姬自动加入可兑换列表

## 六、与配置表的关联

### 6.1 shop.xlsx 配置表结构

| 字段名 | 类型 | 说明 |
|--------|------|------|
| shopId | string | 商品唯一ID |
| shopType | string | 商店类型（sourceCore/mycelium/starCoin/fragment） |
| itemId | string | 关联道具ID（对应items表） |
| itemName | string | 商品名称 |
| cost | int | 价格 |
| currency | string | 货币类型（sourceCore/mycelium/starCoin/r_fragment/sr_fragment/ssr_fragment/ur_fragment） |
| dailyLimit | int | 每日购买上限（0=无限，-1=一次性） |
| sortOrder | int | 商品排序顺序 |
| visible | bool | 是否显示 |
| startTime | string | 上架开始时间（空=永久） |
| endTime | string | 上架结束时间（空=永久） |

### 6.2 数值策划配表要点

1. 每个商品一条记录，`shopType`区分商店类型
2. `dailyLimit`为0表示无限购买，-1表示一次性购买后隐藏
3. 碎片商店的`currency`字段使用碎片类型
4. 礼包类商品在`items`表中配置为礼包类型，包含多个子物品
5. 价格数值需在后续经济平衡中调整
