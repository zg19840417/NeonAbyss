## 一、系统概述

### 1.1 商店入口

商店是庇护所主界面的功能入口之一，位于底部导航栏。

### 1.2 商店类型

| 商店 | 使用货币 | 刷新规则 | 说明 |
|------|---------|---------|------|
| 源核商店 | 源核 | 每日限量 | 购买高级道具 |
| 菌丝商店 | 菌丝 | 每日限量 | 购买基础物资 |
| 星币商店 | 星币 | 每日限量 | 购买付费礼包 |
| 碎片商店 | 碎片 | 不刷新 | 兑换指定角色（详见抽卡策划案） |
| 抽卡（探索） | 源核 | 不刷新 | 单抽/十连抽卡入口（详见抽卡系统策划案） |

### 1.3 刷新规则

- **刷新时间**：每天凌晨5:00（服务器时间）
- **限量说明**：每个商品有每日购买次数上限，刷新后重置
- **刷新提示**：刷新前1小时显示倒计时提示

## 七、与配置表的关联

### 7.1 shop.xlsx 配置表结构

| 字段名 | 类型 | 说明 |
|--------|------|------|
| shopId | string | 商品唯一ID |
| shopType | string | 商店类型（sourceCore/mycelium/starCoin/fragment/gacha） |
| itemId | string | 关联道具ID（对应items表） |
| itemName | string | 商品名称 |
| cost | int | 价格 |
| currency | string | 货币类型（sourceCore/mycelium/starCoin/r_fragment/sr_fragment/ssr_fragment/ur_fragment） |
| dailyLimit | int | 每日购买上限（0=无限，-1=一次性） |
| sortOrder | int | 商品排序顺序 |
| visible | bool | 是否显示 |
| startTime | string | 上架开始时间（空=永久） |
| endTime | string | 上架结束时间（空=永久） |

### 7.2 数值策划配表要点

1. 每个商品一条记录，`shopType`区分商店类型
2. `dailyLimit`为0表示无限购买，-1表示一次性购买后隐藏
3. 碎片商店的`currency`字段使用碎片类型
4. 礼包类商品在`items`表中配置为礼包类型，包含多个子物品
5. 价格数值需在后续经济平衡中调整

### 1. 每日刷新系统

```typescript
// 刷新时间：每天凌晨5:00（服务器时间）
const DAILY_REFRESH_HOUR = 5;

// 刷新逻辑
function checkAndRefreshShop(serverTime: Date, lastRefreshTime: Date): boolean {
  const todayRefresh = new Date(serverTime);
  todayRefresh.setHours(DAILY_REFRESH_HOUR, 0, 0, 0);

  if (serverTime >= todayRefresh && lastRefreshTime < todayRefresh) {
    // 执行刷新
    resetDailyPurchaseLimits();  // 重置所有商品的每日购买计数
    updateLastRefreshTime(serverTime);
    return true;
  }
  return false;
}

// 倒计时计算
function getRefreshCountdown(serverTime: Date): number {
  const nextRefresh = new Date(serverTime);
  nextRefresh.setHours(DAILY_REFRESH_HOUR, 0, 0, 0);
  if (serverTime >= nextRefresh) {
    nextRefresh.setDate(nextRefresh.getDate() + 1);
  }
  return nextRefresh.getTime() - serverTime.getTime(); // 毫秒
}
```

### 4. 免费抗辐射药剂系统

```typescript
// 每1小时自动获得1个免费抗辐射药剂
const FREE_POTION_INTERVAL_MS = 60 * 60 * 1000; // 1小时
const FREE_POTION_STORAGE_MAX = 5;              // 存储上限

interface FreePotionState {
  playerId: string;
  currentCount: number;       // 当前存储数量
  lastGrantTime: number;      // 上次发放时间戳（毫秒）
}

// 检查并发放免费药剂
function checkFreePotion(state: FreePotionState, currentTime: number): number {
  if (state.currentCount >= FREE_POTION_STORAGE_MAX) return 0;

  const elapsed = currentTime - state.lastGrantTime;
  const grantsAvailable = Math.floor(elapsed / FREE_POTION_INTERVAL_MS);

  if (grantsAvailable > 0) {
    const grantsToGive = Math.min(grantsAvailable, FREE_POTION_STORAGE_MAX - state.currentCount);
    state.currentCount += grantsToGive;
    state.lastGrantTime += grantsToGive * FREE_POTION_INTERVAL_MS;
    return grantsToGive;
  }
  return 0;
}

// 领取免费药剂
function claimFreePotion(state: FreePotionState): boolean {
  if (state.currentCount <= 0) return false;
  state.currentCount--;
  // 发放抗辐射药剂道具（与商店购买的共用同一个道具ID）
  grantItem(state.playerId, '抗辐射药剂', 1);
  return true;
}
```

### 6. 抽卡标签页调用 GachaSystem

商店的抽卡标签页不直接处理抽卡逻辑，而是作为入口调用独立的 `GachaSystem` 模块：

```typescript
// 抽卡标签页点击处理
function onGachaTabAction(action: 'single' | 'ten', useTicket: boolean = false): void {
  if (useTicket) {
    // 使用抽卡券
    const ticketItemId = action === 'single' ? 'TICKET_SINGLE' : 'TICKET_TEN';
    if (!InventoryManager.hasItem(ticketItemId, 1)) {
      UIManager.showToast('抽卡券不足');
      return;
    }
    InventoryManager.consumeItem(ticketItemId, 1);
  } else {
    // 使用源核
    const cost = action === 'single' ? 200 : 2000;
    if (!CurrencyManager.hasEnough('sourceCore', cost)) {
      UIManager.showToast('源核不足');
      return;
    }
    CurrencyManager.deduct('sourceCore', cost);
  }

  // 调用 GachaSystem 执行抽卡
  const results = GachaSystem.performGacha(action === 'single' ? 1 : 10);

  // 跳转到抽卡结果展示页
  UIManager.openGachaResult(results);
}
```

**实现要点：**
1. 商店界面仅负责货币/道具扣费和入口展示
2. 实际抽卡概率、保底机制、卡池逻辑等由 `GachaSystem` 独立管理
3. 抽卡结果展示页（动画演出、角色展示等）由 `GachaSystem` 提供的 UI 组件渲染
4. 抽卡标签页需监听 `GachaSystem` 的卡池变更事件，实时更新卡池信息显示
