# 废土元年 — 融合姬养成系统策划案

> **版本**：v1.1
> **版本日期**：2026-04-06
> **用途**：定义融合姬从获取到成型的完整养成路径，涵盖技能升级、能力随机生成、芯片装备、等级消耗等全部养成维度
> **关联文档**：战斗系统策划案、主副卡融合系统策划案（替代原抽卡与合成系统策划案中的合成部分）、职业能力树设计文档、数值配表指导文档
> **v1.1更新**：合成系统由"主副卡融合"更新为"主副卡融合"，重复卡用途新增"作为副卡融合"

---

## 一、系统概述

### 1.1 养成全景图

融合姬的养成分为四个阶段：

```
获取 → 成长 → 强化 → 变体
 │       │       │       │
 ├─抽卡  ├─升级  ├─芯片  ├─异化
 ├─融合  ├─技能  └─芯片  └─额外能力
 ├─碎片  └─能力
 └─N卡赠送
```

### 1.2 核心设计原则

| 原则 | 说明 |
|------|------|
| 品质决定上限 | 品质越高，能力槽越多、基础属性越高、成长空间越大 |
| 随机产生差异 | 同ID的卡，能力组合可能不同，鼓励"刷好胚子" |
| 资源有意义的抉择 | 菌丝同时承担"升级货币"和"合成材料"，玩家需要权衡分配 |
| 长期追求目标 | 满觉醒需16张同名卡，给玩家长期收集动力 |

## 二、获取系统（已有系统，本文仅引用）

融合姬的获取方式在《抽卡与合成系统策划案》（抽卡部分）和《主副卡融合系统策划案》（融合部分）中已完整定义，本文不再重复。核心要点：

| 方式 | 说明 |
|------|------|
| 源核抽卡 | 常驻池 + 限定UP池，三级保底 |
| 主副卡融合 | 1张主卡 + 1张同品质副卡 + 菌丝 → 主卡进化（详见《主副卡融合系统策划案》） |
| 碎片兑换 | 重复卡→碎片，碎片商店兑换指定角色 |
| N卡特殊获取 | 野外首通、禁区事件、任务、成就、新手赠送 |

**重复卡用途（本次更新）**：
- 重复卡**优先用于同名卡的技能升级**（吃同名卡升级技能）
- 重复卡可作为**副卡参与主副卡融合**（改变主卡数值/能力，小概率品质+1）
- 满觉醒且不需要做副卡的重复卡 → 转化为碎片

## 三、等级系统

### 3.1 等级上限

| 规则 | 说明 |
|------|------|
| 等级上限 | = 玩家声望等级 |
| 初始等级 | 获取时为1级 |
| 最大等级 | 99级（声望99级时） |

**示例**：
- 玩家声望30级 → 所有融合姬最高30级
- 玩家声望提升到31级 → 所有融合姬等级上限解锁到31级
- 已满级的融合姬不会自动升级，需要玩家手动消耗菌丝升级

### 3.2 升级消耗

| 项目 | 说明 |
|------|------|
| 升级货币 | **菌丝**（取消"随从经验"概念） |
| 升级方式 | 手动消耗菌丝升级，每级消耗递增 |
| 是否自动 | 不自动，玩家手动操作 |

#### 升级消耗公式

```
第N级升级消耗菌丝 = 基础消耗 × N × 品质系数
```

| 品质 | 品质系数 | 1级→2级消耗 | 10级→11级消耗 | 50级→51级消耗 |
|------|---------|-----------|------------|------------|
| N | 1.0 | 100菌丝 | 1,000菌丝 | 5,000菌丝 |
| R | 1.2 | 120菌丝 | 1,200菌丝 | 6,000菌丝 |
| SR | 1.5 | 150菌丝 | 1,500菌丝 | 7,500菌丝 |
| SSR | 2.0 | 200菌丝 | 2,000菌丝 | 10,000菌丝 |
| UR | 2.5 | 250菌丝 | 2,500菌丝 | 12,500菌丝 |
| LE | 3.0 | 300菌丝 | 3,000菌丝 | 15,000菌丝 |

**设计意图**：
- 高品质卡升级更贵，但基础属性和成长率更高，性价比仍然合理
- 菌丝同时是合成材料，玩家需要权衡"升级现有卡"还是"攒菌丝合成新卡"

### 3.3 属性成长

每次升级提升的三维属性由 `minionCards.xlsx` 中的成长系数决定：

```
当前HP = baseHp + (level - 1) × hpGrowth
当前ATK = baseAtk + (level - 1) × atkGrowth
当前SPD = baseSpd + (level - 1) × spdGrowth
```

### 3.4 等级与声望的关系

| 声望等级 | 解锁内容 | 融合姬等级上限 |
|---------|---------|-------------|
| 1-10 | 地图1（废弃城区） | 1-10级 |
| 11-20 | 地图2（荒野公路） | 11-20级 |
| 21-30 | 地图3（污染森林） | 21-30级 |
| 31-40 | 地图4（干涸河床） | 31-40级 |
| 41-50 | 地图5（军事前哨） | 41-50级 |
| 51-60 | 地图6（沙漠绿洲） | 51-60级 |
| 61-70 | 地图7（军事基地） | 61-70级 |
| 71-80 | 地图8（地下设施） | 71-80级 |
| 81-90 | 地图9（辐射荒原） | 81-90级 |
| 91-99 | 地图10（核心禁区） | 91-99级 |

## 四、技能升级系统（新增）

### 4.1 概述

| 项目 | 说明 |
|------|------|
| 升级方式 | 消耗**同名融合姬卡**（重复卡） |
| 升级规则 | 随机选择一个未满级的主动技能，提升1级 |
| 满级条件 | 3个主动技能全部达到6级 |
| 满觉醒所需 | 1张本体 + 15张同名卡 = **16张同名卡** |

### 4.2 技能等级

| 项目 | 说明 |
|------|------|
| 初始等级 | 所有技能默认1级 |
| 最大等级 | 6级 |
| 升级次数 | 每个技能最多升5次 |
| 总升级次数 | 3个技能 × 5次 = 15次 |

### 4.3 升级流程

```
玩家选择一张融合姬作为"本体"
    ↓
消耗1张同名卡（从背包中选择）
    ↓
系统从本体所有未满级的主动技能中随机选择1个
    ↓
该技能等级+1
    ↓
根据技能表配置，应用该等级的升级效果
    ↓
如果所有技能都已满级 → 提示"该融合姬已满觉醒，无法继续升级"
    ↓
满觉醒后的同名卡 → 自动转化为碎片
```

### 4.4 升级效果配置

技能升级的效果**不是固定公式**，而是由 `skills.xlsx` 中的配置决定。每个技能可以有不同的升级方向：

| 升级方向 | 配置字段 | 示例 |
|---------|---------|------|
| 倍率提升 | `levelUpMultiplierBonus` | 每级+0.1倍率（技能2基础2.0→满级2.5） |
| 冷却减少 | `levelUpCooldownReduce` | 每级-0.5回合冷却（技能3基础8回合→满级5.5回合） |
| 附加效果增强 | `levelUpEffectBoost` | 每级附加效果数值+10% |
| 目标数增加 | `levelUpTargetBoost` | 满级时单体技能变为群体 |
| 特殊效果解锁 | `levelUpUnlockEffect` | 3级解锁附加燃烧、5级解锁附加眩晕 |

**配置示例**（skills.xlsx 新增字段）：

```json
// 技能：锈叶鞭击（技能2）
{
  "skillId": "SK_XUYE_002",
  "name": "锈叶鞭击",
  "baseMultiplier": 2.0,
  "cooldown": 4,
  "maxLevel": 6,
  "levelUpBonus": [
    {"level": 2, "multiplierBonus": 0.1},
    {"level": 3, "multiplierBonus": 0.1, "unlockEffect": {"type":"bleed","value":0.05,"duration":2}},
    {"level": 4, "multiplierBonus": 0.1},
    {"level": 5, "multiplierBonus": 0.1, "effectBoost": {"type":"bleed","valueBonus":0.05}},
    {"level": 6, "multiplierBonus": 0.15}
  ]
}
// 满级效果：倍率2.0→2.55，冷却不变，附加流血0.05→0.10
```

### 4.5 普攻升级

普攻（技能1）**参与升级池**，吃同名卡时可能升级普攻。普攻的升级效果通常为：

| 等级 | 典型升级效果 |
|------|------------|
| 1→2 | 倍率+0.05 |
| 2→3 | 倍率+0.05 |
| 3→4 | 倍率+0.05，解锁附加效果 |
| 4→5 | 倍率+0.05，附加效果增强 |
| 5→6 | 倍率+0.1（满级大跳跃） |

### 4.6 满觉醒标识

当一张融合姬的3个技能全部达到6级时：

| 变化 | 说明 |
|------|------|
| 卡牌标识 | 卡面增加"满觉醒"标识/特效 |
| 能力槽 | 不额外增加（能力槽数量仍由品质决定） |
| 后续同名卡 | 自动转化为对应品质碎片 |

## 五、能力（被动）系统

### 5.1 能力生成规则

能力在**融合姬卡牌生成时随机抽取**，而非固定配置。

| 项目 | 说明 |
|------|------|
| 生成时机 | 抽卡获得、合成产出、碎片兑换时 |
| 数量 | 随机，但不超过品质对应的槽上限 |
| 内容 | 从配表定义的能力池中按权重随机抽取 |
| 必带能力 | 支持"必定生成某能力"的设定（权重=强制） |

### 5.2 能力槽数量（按品质）

| 品质 | 槽上限 | 说明 |
|------|--------|------|
| N | 2 | 1个职业能力 + 1个种族/元素能力 |
| R | 3 | 1个职业能力 + 1个种族/元素能力 + 1个随机能力 |
| SR | 4 | 1个职业能力 + 1个种族/元素能力 + 2个随机能力 |
| SSR | 4 | 1个职业核心能力 + 1个种族/元素能力 + 2个随机能力 |
| UR | 5 | 1个职业核心能力 + 1个种族/元素能力 + 3个随机能力 |
| LE | 6 | 1个职业终极能力 + 1个种族/元素能力 + 4个随机能力 |

**注意**：槽数量是上限，实际生成时可能少于上限（低概率）。

### 5.3 能力来源与选取规则

| 来源 | 占槽 | 选取规则 |
|------|------|---------|
| 职业能力 | 必占1槽 | 根据品质从职业通用/核心/终极能力池中选取 |
| 种族能力 | 必占1槽 | 根据种族从对应种族能力池中选取 |
| 元素能力 | 与种族二选一 | 根据元素从对应元素能力池中选取 |
| 随机能力 | 占剩余槽 | 从通用能力池或职业方向能力树中按权重随机选取 |

**种族/元素能力二选一规则**：
- 50%概率选取种族能力，50%概率选取元素能力
- 可通过配表强制指定（如某张卡必定带种族能力）

### 5.4 能力池配置

能力池的详细定义见《职业能力树设计文档》，本文仅说明配表方式。

每张融合姬卡在 `minionCards.xlsx` 中新增以下字段：

| 字段 | 类型 | 说明 |
|------|------|------|
| `abilityPoolId` | string | 引用能力池配置表，定义该卡可以从哪些能力中随机 |
| `forcedAbilities` | array | 必定生成的能力ID列表（如 `["TANK_CORE_01"]`） |
| `abilityWeights` | json | 各能力来源的权重配置 |

**配置示例**：

```json
// SSR品质坦克 - 苔甲·叶青
{
  "cardId": "MC_water_tank_004",
  "abilityPoolId": "POOL_MC_water_tank_004",
  "forcedAbilities": ["TANK_CORE_01"],
  "abilityWeights": {
    "class": {"pool": "TANK_CORE", "count": 1, "forced": ["TANK_CORE_01"]},
    "race": {"pool": "RACE_PLANT", "count": 1, "weight": 0.5},
    "element": {"pool": "ELEM_WATER", "count": 1, "weight": 0.5},
    "random": {"pool": "TANK_A+TANK_B+TANK_C", "count": 2, "weight": "per_ability"}
  }
}
```

### 5.5 能力池配置表（新建 abilities.xlsx）

| 字段名 | 类型 | 说明 |
|--------|------|------|
| abilityId | string | 能力唯一ID（如 "TANK_DEF_01"） |
| name | string | 能力名称 |
| description | string | 能力描述（精确到数值） |
| type | string | 能力类型：buff / debuff / trigger / heal / special |
| target | string | 作用目标：self / ally_all / ally_lowest_hp / ally_single / enemy_all / enemy_single |
| condition | string | 触发条件：always / battle_start / turn_start / on_hit / on_crit / on_kill / on_attacked / on_ally_attacked / on_ally_hp_below_30% / on_ally_hp_below_20% / on_self_hp_below_30% / on_self_hp_below_50% / on_self_death / on_ally_death / on_enemy_hp_below_30% / after_normal_attack / after_ultimate |
| effect | string | 效果类型：damage_reduce_percent / damage_reduce_fixed / dodge / crit_rate_boost / crit_damage_boost / atk_boost_percent / spd_boost_percent / hp_boost_percent / heal_percent / heal_maxhp_percent / lifesteal / reflect_damage / shield / taunt / true_damage_convert / ignore_damage_reduce 等 |
| value | float | 效果数值（百分比用小数，如 0.1 = 10%） |
| duration | int | 持续回合数（0=永久） |
| stackable | bool | 是否可叠加 |
| maxStack | int | 最大叠加层数（0=不可叠加） |
| source | string | 来源：class / race / element / random / mutation |
| qualityMin | string | 最低品质要求：N / R / SR / SSR / UR / LE |
| qualityMax | string | 最高品质限制：N / R / SR / SSR / UR / LE（null=不限） |

### 5.6 能力冲突检查

以下能力**不可同时配置**在同一角色上：

| 冲突组 | 原因 |
|--------|------|
| TANK_DEF_01 + TANK_DEF_02 | 同类型减伤，取最高不叠加 |
| DPS_CRIT_01 + DPS_CRIT_02 | 同类型暴击率，取最高不叠加 |
| SUP_HEAL_01 + SUP_HEAL_02 | 同类型治疗，取最高不叠加 |
| 不同种族能力 | 只能有1个种族能力 |
| 不同元素能力 | 只能有1个元素能力 |

### 5.7 异化额外能力

| 项目 | 说明 |
|------|------|
| 触发条件 | SSR品质 + 异化改造晶片合成 |
| 占槽 | **不占槽**，纯额外奖励 |
| 获取概率 | 70%无额外 / 25%获得1个被动 / 5%获得1个主动 |
| 存储 | 通过 `mutationAbilityId` 字段单独存储 |
| 来源池 | 从通用能力池中随机选取，不与已有能力冲突 |

## 六、芯片系统（整合）

### 6.1 系统整合说明

**取消装备卡系统**，只保留强化芯片作为唯一的队伍强化手段。

| 项目 | 说明 |
|------|------|
| 装备位置 | 每支队伍携带**1张**强化芯片 |
| 作用范围 | 全队3个融合姬 |
| 加成方式 | HP% + ATK% + 被动技能 |
| 合成规则 | 3张同品质芯片 → 1张随机更高品质芯片（不消耗菌丝） |

### 6.2 芯片加成计算

```
最终HP = 融合姬当前HP × (1 + 芯片 hpPercent)
最终ATK = 融合姬当前ATK × (1 + 芯片 atkPercent)
```

### 6.3 芯片技能数量（按品质）

| 品质 | 技能数量 | 说明 |
|------|---------|------|
| N | 1个 | 基础光环 |
| R | 1个 | 基础光环 |
| SR | 1个 | 中级光环 |
| SSR | 2个 | 中级光环 + 触发型被动 |
| UR | 2个 | 高级光环 + 触发型被动 |
| LE | 3个 | 高级光环 + 触发型被动 + 改变技能 |

## 七、经济循环

### 7.1 菌丝流向

```
菌丝获取                          菌丝消耗
├─ 野外关卡通关奖励               ├─ 融合姬升级（每级消耗）
├─ 禁区事件奖励                   ├─ 主副卡融合
├─ 商店兑换（碎片→菌丝）          └─ （未来可扩展）
├─ 每日任务奖励
└─ 成就奖励
```

### 7.2 同名卡流向

```
同名卡获取                         同名卡消耗
├─ 抽卡重复                       ├─ 技能升级（每次消耗1张）
├─ 融合产出重复                   ├─ 作为副卡融合（每次消耗1张）
└─ 碎片商店兑换                   └─ 多余 → 转化为碎片
```

### 7.3 碎片流向

```
碎片获取                           碎片消耗
├─ 重复卡转化（满觉醒后）          ├─ 碎片商店兑换指定角色
└─ （无其他来源）                  └─ 兑换改造晶片/菌丝/源核
```

## 八、养成进度总览

### 8.1 单张融合姬的养成里程碑

| 阶段 | 条件 | 收益 |
|------|------|------|
| 获取 | 抽卡/合成/兑换获得 | 1级，初始能力随机生成 |
| 培养中 | 消耗菌丝升级到声望等级上限 | 三维属性达到当前上限 |
| 技能强化 | 消耗同名卡升级技能 | 技能效果逐步增强 |
| 满觉醒 | 3个技能全部6级 | 获得满觉醒标识 |
| 异化（可选） | SSR + 异化改造晶片 | 属性×120%，可能获得额外能力 |

### 8.2 一张SSR卡从获取到满觉醒的成本估算

| 项目 | 消耗 |
|------|------|
| 获取1张SSR | 约40抽（保底）= 8,000源核 |
| 升级到50级（声望50级） | 约75,000菌丝（累计） |
| 满觉醒（+15张同名SSR） | 约600抽（期望）= 120,000源核 |
| **总计** | **约128,000源核 + 75,000菌丝** |

> 注：以上为粗略估算，实际成本受保底机制和运气影响。

#### SkillUpgradeSystem.js — 技能升级系统

```javascript
class SkillUpgradeSystem {
    constructor() {
        this.qualityCoefficients = {
            N: 1.0, R: 1.2, SR: 1.5, SSR: 2.0, UR: 2.5, LE: 3.0
        };
    }

    /**
     * 消耗同名卡升级技能
     * @param {string} cardInstanceId - 本体卡的实例ID
     * @param {string} materialCardInstanceId - 材料卡的实例ID
     * @returns {Object} { success, skillId, oldLevel, newLevel, newEffect }
     */
    upgradeSkill(cardInstanceId, materialCardInstanceId) {
        const card = inventory.getCard(cardInstanceId);
        const material = inventory.getCard(materialCardInstanceId);

        // 1. 验证同名
        if (card.cardId !== material.cardId) {
            throw new Error('必须使用同名卡进行升级');
        }

        // 2. 获取未满级技能列表
        const unmaxedSkills = card.skills.filter(s => s.currentLevel < s.maxLevel);
        if (unmaxedSkills.length === 0) {
            throw new Error('所有技能已满级，该融合姬已满觉醒');
        }

        // 3. 随机选择一个技能
        const targetSkill = unmaxedSkills[Math.floor(Math.random() * unmaxedSkills.length)];

        // 4. 升级
        const oldLevel = targetSkill.currentLevel;
        targetSkill.currentLevel += 1;

        // 5. 应用升级效果
        const levelUpBonus = this.getLevelUpBonus(targetSkill.skillId, targetSkill.currentLevel);
        this.applyLevelUpBonus(targetSkill, levelUpBonus);

        // 6. 消耗材料卡
        inventory.removeCard(materialCardInstanceId);

        // 7. 检查是否满觉醒
        const isMaxAwakened = card.skills.every(s => s.currentLevel >= s.maxLevel);

        return {
            success: true,
            skillId: targetSkill.skillId,
            skillName: targetSkill.name,
            oldLevel,
            newLevel: targetSkill.currentLevel,
            newEffect: levelUpBonus,
            isMaxAwakened
        };
    }

    /**
     * 获取指定等级的升级效果
     */
    getLevelUpBonus(skillId, level) {
        const skillConfig = ConfigManager.getSkillById(skillId);
        if (!skillConfig.levelUpBonus) return null;

        return skillConfig.levelUpBonus.find(b => b.level === level);
    }

    /**
     * 应用升级效果
     */
    applyLevelUpBonus(skill, bonus) {
        if (!bonus) return;

        if (bonus.multiplierBonus) {
            skill.currentMultiplier += bonus.multiplierBonus;
        }
        if (bonus.cooldownReduce) {
            skill.currentCooldown -= bonus.cooldownReduce;
        }
        if (bonus.unlockEffect) {
            skill.additionalEffects.push(bonus.unlockEffect);
        }
        if (bonus.effectBoost) {
            // 增强已有附加效果
            const target = skill.additionalEffects.find(
                e => e.type === bonus.effectBoost.type
            );
            if (target && bonus.effectBoost.valueBonus) {
                target.value += bonus.effectBoost.valueBonus;
            }
        }
        if (bonus.targetBoost) {
            skill.currentTargetType = bonus.targetBoost;
        }
    }

    /**
     * 计算觉醒进度
     */
    getAwakenProgress(cardInstanceId) {
        const card = inventory.getCard(cardInstanceId);
        const totalLevels = card.skills.reduce((sum, s) => sum + (s.maxLevel - 1), 0);
        const currentLevels = card.skills.reduce((sum, s) => sum + (s.currentLevel - 1), 0);
        return { current: currentLevels, total: totalLevels, percent: currentLevels / totalLevels };
    }
}
```

#### AbilityGenerator.js — 能力随机生成器

```javascript
class AbilityGenerator {
    /**
     * 为融合姬生成能力
     * @param {Object} cardConfig - minionCards表中的配置
     * @returns {string[]} abilityIds
     */
    generate(cardConfig) {
        const abilities = [];
        const quality = cardConfig.quality;
        const maxSlots = this.getMaxSlots(quality);

        // 1. 职业能力（必占1槽）
        const classAbility = this.selectClassAbility(cardConfig);
        if (classAbility) abilities.push(classAbility);

        // 2. 种族/元素能力（必占1槽，二选一）
        const raceOrElementAbility = this.selectRaceOrElementAbility(cardConfig);
        if (raceOrElementAbility) abilities.push(raceOrElementAbility);

        // 3. 必带能力
        if (cardConfig.forcedAbilities) {
            for (const forcedId of cardConfig.forcedAbilities) {
                if (!abilities.includes(forcedId)) {
                    abilities.push(forcedId);
                }
            }
        }

        // 4. 随机能力（填充剩余槽）
        const remainingSlots = maxSlots - abilities.length;
        if (remainingSlots > 0) {
            const randomAbilities = this.selectRandomAbilities(cardConfig, remainingSlots, abilities);
            abilities.push(...randomAbilities);
        }

        // 5. 冲突检查
        this.checkConflicts(abilities);

        return abilities;
    }

    /**
     * 获取品质对应的最大能力槽
     */
    getMaxSlots(quality) {
        const slots = { N: 2, R: 3, SR: 4, SSR: 4, UR: 5, LE: 6 };
        return slots[quality] || 2;
    }

    /**
     * 选取职业能力
     */
    selectClassAbility(cardConfig) {
        // 如果有必带职业能力，直接返回
        if (cardConfig.forcedAbilities) {
            const classForced = cardConfig.forcedAbilities.find(id => id.startsWith('TANK_') || id.startsWith('DPS_') || id.startsWith('SUP_'));
            if (classForced) return classForced;
        }

        // 从职业能力池中按品质选取
        const quality = cardConfig.quality;
        let poolType;
        if (quality === 'LE') poolType = 'ULT';
        else if (['SSR', 'UR'].includes(quality)) poolType = 'CORE';
        else poolType = 'BASE';

        const pool = ConfigManager.getAbilitiesBySourceAndType('class', poolType, cardConfig.profession);
        return this.weightedRandom(pool);
    }

    /**
     * 选取种族或元素能力（二选一）
     */
    selectRaceOrElementAbility(cardConfig) {
        const weights = cardConfig.abilityWeights || {};

        const raceWeight = weights.race?.weight || 0.5;
        const elementWeight = weights.element?.weight || 0.5;

        if (Math.random() < raceWeight / (raceWeight + elementWeight)) {
            // 选取种族能力
            const pool = ConfigManager.getAbilitiesBySourceAndType('race', null, cardConfig.race);
            return this.weightedRandom(pool);
        } else {
            // 选取元素能力
            const pool = ConfigManager.getAbilitiesBySourceAndType('element', null, cardConfig.element);
            return this.weightedRandom(pool);
        }
    }

    /**
     * 选取随机能力
     */
    selectRandomAbilities(cardConfig, count, existingAbilities) {
        const results = [];
        const pool = ConfigManager.getAbilitiesBySource('random');

        for (let i = 0; i < count; i++) {
            // 过滤掉已选的和冲突的
            const available = pool.filter(a =>
                !results.includes(a.abilityId) &&
                !existingAbilities.includes(a.abilityId) &&
                !this.isConflicting(a.abilityId, [...results, ...existingAbilities]) &&
                this.meetsQualityRequirement(a, cardConfig.quality)
            );

            if (available.length > 0) {
                const selected = this.weightedRandom(available);
                results.push(selected.abilityId);
            }
        }

        return results;
    }

    /**
     * 冲突检查
     */
    checkConflicts(abilityIds) {
        const conflictGroups = [
            ['TANK_DEF_01', 'TANK_DEF_02'],
            ['DPS_CRIT_01', 'DPS_CRIT_02'],
            ['SUP_HEAL_01', 'SUP_HEAL_02']
        ];

        for (const group of conflictGroups) {
            const count = abilityIds.filter(id => group.includes(id)).length;
            if (count > 1) {
                console.warn(`能力冲突: ${group.join(' 和 ')} 不能同时存在`);
            }
        }

        const raceAbilities = abilityIds.filter(id => id.startsWith('RACE_'));
        if (raceAbilities.length > 1) {
            console.warn('只能有1个种族能力');
        }

        const elemAbilities = abilityIds.filter(id => id.startsWith('ELEM_'));
        if (elemAbilities.length > 1) {
            console.warn('只能有1个元素能力');
        }
    }

    /**
     * 检查品质要求
     */
    meetsQualityRequirement(ability, cardQuality) {
        const qualityRank = { N: 1, R: 2, SR: 3, SSR: 4, UR: 5, LE: 6 };
        const cardRank = qualityRank[cardQuality];
        const minRank = ability.qualityMin ? qualityRank[ability.qualityMin] : 1;
        const maxRank = ability.qualityMax ? qualityRank[ability.qualityMax] : 6;

        return cardRank >= minRank && cardRank <= maxRank;
    }

    /**
     * 加权随机
     */
    weightedRandom(items) {
        const totalWeight = items.reduce((sum, item) => sum + (item.weight || 1), 0);
        let random = Math.random() * totalWeight;

        for (const item of items) {
            random -= (item.weight || 1);
            if (random <= 0) return item.abilityId || item;
        }

        return items[items.length - 1].abilityId || items[items.length - 1];
    }

    /**
     * 检查是否冲突
     */
    isConflicting(abilityId, existingIds) {
        const conflictGroups = [
            ['TANK_DEF_01', 'TANK_DEF_02'],
            ['DPS_CRIT_01', 'DPS_CRIT_02'],
            ['SUP_HEAL_01', 'SUP_HEAL_02']
        ];

        for (const group of conflictGroups) {
            if (group.includes(abilityId) && existingIds.some(id => group.includes(id))) {
                return true;
            }
        }

        if (abilityId.startsWith('RACE_') && existingIds.some(id => id.startsWith('RACE_'))) {
            return true;
        }

        if (abilityId.startsWith('ELEM_') && existingIds.some(id => id.startsWith('ELEM_'))) {
            return true;
        }

        return false;
    }
}
```

#### LevelUpSystem.js — 融合姬升级系统

```javascript
class LevelUpSystem {
    constructor() {
        this.qualityCoefficients = {
            N: 1.0, R: 1.2, SR: 1.5, SSR: 2.0, UR: 2.5, LE: 3.0
        };
    }

    /**
     * 计算升级消耗
     * @param {int} currentLevel - 当前等级
     * @param {string} quality - 品质
     * @returns {int} 消耗的菌丝数量
     */
    calcUpgradeCost(currentLevel, quality) {
        const baseCost = ConfigManager.getLevelUpBaseCost(currentLevel + 1);
        const coefficient = this.qualityCoefficients[quality] || 1.0;
        return Math.floor(baseCost * currentLevel * coefficient);
    }

    /**
     * 升级融合姬
     * @param {string} cardInstanceId - 卡牌实例ID
     * @param {int} count - 升级次数（默认1）
     * @returns {Object} { success, oldLevel, newLevel, totalCost, statGains }
     */
    levelUp(cardInstanceId, count = 1) {
        const card = inventory.getCard(cardInstanceId);
        const playerLevel = PlayerData.reputationLevel;

        // 检查等级上限
        if (card.level >= playerLevel) {
            throw new Error(`融合姬等级已达上限（${playerLevel}级），请先提升声望等级`);
        }

        let totalCost = 0;
        let actualCount = 0;
        const statGains = { hp: 0, atk: 0, spd: 0 };

        for (let i = 0; i < count; i++) {
            if (card.level >= playerLevel) break;

            const cost = this.calcUpgradeCost(card.level, card.quality);
            if (inventory.getMycelium() < totalCost + cost) break;

            totalCost += cost;
            actualCount++;

            // 计算属性增长
            statGains.hp += card.hpGrowth;
            statGains.atk += card.atkGrowth;
            statGains.spd += card.spdGrowth;

            card.level += 1;
        }

        if (actualCount === 0) {
            throw new Error('菌丝不足');
        }

        // 扣除菌丝
        inventory.removeMycelium(totalCost);

        return {
            success: true,
            oldLevel: card.level - actualCount,
            newLevel: card.level,
            totalCost,
            actualCount,
            statGains
        };
    }

    /**
     * 批量升级到指定等级
     */
    levelUpTo(cardInstanceId, targetLevel) {
        const card = inventory.getCard(cardInstanceId);
        const count = Math.min(targetLevel, PlayerData.reputationLevel) - card.level;
        if (count <= 0) {
            throw new Error('目标等级无效');
        }
        return this.levelUp(cardInstanceId, count);
    }
}
```

```javascript
// 旧逻辑：重复卡直接转碎片
// 新逻辑：优先用于技能升级，满觉醒后才转碎片

handleDuplicateCard(cardInstanceId) {
    const card = inventory.getCard(cardInstanceId);
    const sameCards = inventory.getCardsByCardId(card.cardId);

    // 找到同cardId中未满觉醒的本体
    const mainCard = sameCards.find(c => {
        return !SkillUpgradeSystem.isMaxAwakened(c);
    });

    if (mainCard) {
        // 有未满觉醒的本体 → 提示可用于技能升级
        return {
            type: 'skill_upgrade_available',
            mainCardInstanceId: mainCard.instanceId,
            message: '检测到同名融合姬，可用于技能升级'
        };
    } else {
        // 所有同名卡已满觉醒 → 转化为碎片
        return FragmentSystem.convertToFragment(card);
    }
}
```
