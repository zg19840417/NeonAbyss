## 一、系统概述

### 1.1 设计目的

新手引导是玩家首次进入《废土元年》后的核心体验环节。通过7步引导流程，帮助玩家快速理解游戏核心玩法循环（编队→战斗→探索→资源获取），降低流失率，提升首日留存。

### 1.2 设计原则

| 原则 | 说明 |
|------|------|
| **强制引导** | 前4步为核心战斗引导，不可跳过，确保玩家完成首次战斗 |
| **渐进式教学** | 每步只教授一个核心功能，避免信息过载 |
| **即时奖励** | 每步引导完成后立即发放奖励，建立正向反馈 |
| **看板娘陪伴** | 全程由看板娘"芽衣"进行对话引导，增强代入感 |
| **可回溯** | 引导完成后可在设置中重新查看引导内容 |

### 1.3 引导触发条件

- **首次登录**：玩家创建角色后自动触发
- **回归玩家**：若玩家完成引导后超过7天未登录，再次登录时弹出"回归引导"提示（非强制）
- **版本更新**：新增系统时，通过顶部横幅提示玩家查看新功能引导

### 1.4 引导状态管理

```
GuideState {
    currentStep: int          // 当前引导步骤 (0=未开始, 1-7=进行中, 8=已完成)
    stepStartTime: long       // 当前步骤开始时间戳
    totalGuideTime: long      // 引导总耗时
    isSkippable: bool         // 当前步骤是否可跳过
    completedSteps: int[]     // 已完成的步骤列表
    skipCount: int            // 跳过次数（用于数据分析）
    lastLoginDate: string     // 上次登录日期
    isReturningPlayer: bool   // 是否为回归玩家
}
```

## 二、引导流程（7步详细设计）

### 2.1 第一步：领取初始角色

| 项目 | 内容 |
|------|------|
| **步骤ID** | guide_step_01 |
| **触发条件** | 进入主界面后自动触发 |
| **是否可跳过** | 否（强制） |
| **预计耗时** | 30秒 |

#### 引导内容

**看板娘对话（逐条显示，每条停留2秒）：**

> **芽衣**：「指挥官，你终于醒了！欢迎来到废土元年……这里的世界已经面目全非了。」
>
> **芽衣**：「别担心，我是你的向导芽衣。在庇护所里，你不会孤单的。」
>
> **芽衣**：「废土上危机四伏，你需要组建自己的战斗小队才能生存下去。」
>
> **芽衣**：「来吧，先看看你的初始伙伴们！点击底部的【队伍】按钮。」

**UI操作指引：**

- 底部导航栏【队伍】按钮高亮闪烁（金色光圈动画，频率1Hz）
- 其余5个导航按钮添加半透明黑色遮罩（opacity: 0.5）
- 看板娘位于屏幕右侧，对话气泡从右侧滑入

**完成条件：**
- 玩家点击底部导航栏【队伍】按钮

**完成奖励：**
- 3张N品质融合姬卡：铁壁藤蔓（坦克/植物系）、火花蜂（输出/动物系）、荧光苔（辅助/植物系）
- 1张N品质强化芯片：基础防御芯片
- 系统自动弹出"获得新伙伴"展示界面，逐张展示卡牌立绘与属性

**奖励发放动画：**
1. 屏幕中央弹出卡牌获得动画（卡牌从光柱中升起，持续1.5秒/张）
2. 卡牌展示顺序：铁壁藤蔓 → 火花蜂 → 荧光苔 → 基础防御芯片
3. 每张卡展示完毕后，右上角弹出"+1"提示

**异常处理：**
- 若玩家点击了非目标区域，看板娘提示：「指挥官，请点击底部的【队伍】按钮哦～」
- 若玩家10秒未操作，目标按钮闪烁频率加快至2Hz

### 2.2 第二步：编辑队伍

| 项目 | 内容 |
|------|------|
| **步骤ID** | guide_step_02 |
| **触发条件** | 第一步完成后自动触发 |
| **是否可跳过** | 否（强制） |
| **预计耗时** | 45秒 |

#### 引导内容

**看板娘对话：**

> **芽衣**：「太好了！这些就是你的初始伙伴。铁壁藤蔓可以承受伤害，火花蜂负责输出，荧光苔能治疗队友。」
>
> **芽衣**：「现在你需要将她们编入战斗队伍。点击【编队】按钮来组建你的第一支小队吧！」

**UI操作指引：**

- 队伍界面右上角【编队】按钮高亮闪烁
- 融合姬列表区域添加半透明遮罩（引导完成后解除）
- 编队界面中3个融合姬槽位 + 1个芯片槽位用虚线框标注

**编队流程：**

1. 玩家点击【编队】→ 进入编队界面
2. 系统自动将3张融合姬卡和1张芯片卡拖入对应槽位（新手引导期间自动编入）
3. 编入完成后，槽位边框变为实线并播放确认动画

**看板娘对话（编队完成后）：**

> **芽衣**：「完美！你的第一支小队组建完成了。铁壁藤蔓在前排抗伤，火花蜂在后排输出，荧光苔负责治疗，再配上防御芯片……虽然只是初始配置，但足以应对废土上的小威胁了。」

**完成条件：**
- 3张融合姬卡 + 1张芯片卡全部编入队伍槽位

**完成奖励：**
- 5000菌丝（货币）
- 1000源核（货币）
- 奖励通过飘字动画显示在屏幕中央

**异常处理：**
- 若编队界面中卡牌不足（理论上不会发生），系统自动补发缺失卡牌
- 若玩家尝试编入非推荐卡牌，允许操作但不影响引导进度

### 2.4 第四步：完成首次战斗

| 项目 | 内容 |
|------|------|
| **步骤ID** | guide_step_04 |
| **触发条件** | 第三步完成后自动触发 |
| **是否可跳过** | 否（强制） |
| **预计耗时** | 60秒 |

#### 引导内容

**战斗前对话：**

> **芽衣**：「遭遇敌人了！别紧张，战斗是自动进行的，你只需要在关键时刻释放技能就好。」
>
> **芽衣**：「当融合姬的技能能量充满时，头像会发光。这时候点击头像就能释放强力技能！」

**战斗引导机制：**

- **敌人配置**：3只N品质变异老鼠（HP: 200/只，攻击力: 30/只）
- **我方配置**：铁壁藤蔓(HP: 800) + 火花蜂(HP: 500) + 荧光苔(HP: 400) + 基础防御芯片
- **战斗模式**：自动战斗 + 手动技能释放
- **技能引导**：战斗开始15秒后，火花蜂技能能量充满，头像发光并弹出提示「点击释放技能！」

**战斗中引导提示：**

1. **第5秒**：屏幕顶部弹出提示「融合姬会自动攻击敌人」
2. **第10秒**：提示「注意观察技能能量条」
3. **第15秒**：火花蜂头像发光，弹出手指动画指向头像，提示「技能就绪！点击火花蜂头像释放技能」
4. **若玩家5秒内未点击**：系统自动释放技能，并提示「下次记得手动释放哦，技能伤害会更高！」

**战斗结算：**

- 战斗胜利后进入结算界面
- 显示：战斗评价（S级）、获得经验、获得物品
- 结算界面停留3秒后自动关闭

**战斗后对话：**

> **芽衣**：「漂亮！第一次战斗就拿到了S级评价！看来你很有天赋呢。」
>
> **芽衣**：「随着探索深入，你会遇到更强的敌人和更丰厚的奖励。」

**完成条件：**
- 战斗胜利并完成结算

**完成奖励：**
- 3抗辐射药剂（消耗品）
- 500经验值（全队共享）
- 100菌丝
- 奖励通过结算界面展示

## 三、强制引导规则

### 3.1 强制引导范围

| 步骤 | 是否强制 | 原因 |
|------|---------|------|
| 第1步：领取初始角色 | 强制 | 玩家必须拥有初始卡牌才能进行后续操作 |
| 第2步：编辑队伍 | 强制 | 玩家必须组建队伍才能进行战斗 |
| 第3步：进入探索 | 强制 | 玩家必须了解探索入口 |
| 第4步：完成首次战斗 | 强制 | 玩家必须体验核心战斗玩法 |
| 第5步：查看商店 | 非强制 | 商店为辅助功能，可后续自行了解 |
| 第6步：查看任务 | 非强制 | 任务系统有独立入口提示 |
| 第7步：互动结束 | 非强制 | 庆祝环节，可快速跳过 |

### 3.2 强制引导限制

在强制引导期间（第1-4步），以下操作被禁止：

- **禁止切换底部导航**：除引导目标外的导航按钮不可点击
- **禁止打开侧边栏/设置**：所有非引导相关的UI入口被屏蔽
- **禁止关闭游戏**：若玩家最小化游戏，重新打开后从当前步骤继续
- **禁止跳过对话**：强制步骤中的看板娘对话不可跳过，必须等待播放完毕
- **禁止加速战斗**：首次战斗不支持加速/跳过功能

### 3.3 引导中断处理

| 中断场景 | 处理方式 |
|---------|---------|
| 应用切到后台 | 重新打开后从当前步骤继续，对话重新播放 |
| 网络断开 | 提示"网络连接中断"，恢复后从当前步骤继续 |
| 崩溃重启 | 从最近完成的步骤之后继续（服务端记录进度） |
| 数据清除 | 重新开始全部引导流程 |

## 四、引导跳过机制

### 4.1 跳过按钮规则

- **显示时机**：第5步开始时出现
- **按钮位置**：屏幕右上角，与关闭按钮同区域
- **按钮样式**：白色文字「跳过引导」+ 灰色底色，字号12px
- **点击效果**：弹出二次确认弹窗

### 4.2 跳过确认弹窗

```
┌─────────────────────────────┐
│                             │
│    确定要跳过新手引导吗？    │
│                             │
│  跳过后你仍可在设置中重新    │
│  查看引导内容。              │
│                             │
│  ┌──────────┐ ┌──────────┐  │
│  │  继续引导  │ │  确认跳过  │  │
│  └──────────┘ └──────────┘  │
│                             │
└─────────────────────────────┘
```

### 4.3 跳过后的处理

- 立即将引导状态标记为已完成
- 发放所有已解锁步骤的累计奖励
- 解锁全部UI功能
- 在设置→帮助→新手引导中保留回看入口

### 4.4 回归玩家引导

| 条件 | 行为 |
|------|------|
| 离线 < 3天 | 正常进入游戏，无额外引导 |
| 离线 3-7天 | 登录时弹出"版本更新日志"横幅（非强制） |
| 离线 7-30天 | 弹出"欢迎回来"弹窗，包含近期更新摘要 + 一键领取回归奖励 |

```csharp
// TutorialManager.cs
public class TutorialManager : MonoBehaviour
{
    public static TutorialManager Instance { get; private set; }

    private TutorialConfig currentConfig;
    private int currentStep = 0;
    private bool isGuideActive = false;

    /// <summary>
    /// 初始化引导系统，检查是否需要启动引导
    /// </summary>
    public void Initialize()
    {
        var saveData = TutorialData.LoadProgress();
        if (saveData.currentStep == 0)
        {
            StartStep(1);
        }
        else if (saveData.currentStep < 8)
        {
            // 断点续传
            StartStep(saveData.currentStep);
        }
    }

    /// <summary>
    /// 开始指定步骤
    /// </summary>
    public void StartStep(int stepId)
    {
        currentStep = stepId;
        isGuideActive = true;
        currentConfig = TutorialConfigTable.Get(stepId);

        // 发送埋点
        AnalyticsManager.Track("tutorial_step_start", stepId);

        // 显示遮罩
        TutorialUI.ShowMask(currentConfig.highlight_area);

        // 播放对话
        StartCoroutine(PlayDialogues(currentConfig.dialogue_list));

        // 注册完成条件监听
        RegisterCompletionListener(currentConfig.completion_condition);
    }

    /// <summary>
    /// 完成当前步骤
    /// </summary>
    public void CompleteStep(int stepId)
    {
        // 发放奖励
        var rewards = TutorialRewardTable.GetByStep(stepId);
        foreach (var reward in rewards)
        {
            RewardManager.GiveReward(reward);
        }

        // 更新进度
        TutorialData.SaveProgress(stepId + 1);

        // 发送埋点
        AnalyticsManager.Track("tutorial_step_complete", stepId);

        // 进入下一步
        if (stepId < 7)
        {
            StartStep(stepId + 1);
        }
        else
        {
            FinishGuide();
        }
    }

    /// <summary>
    /// 跳过引导（仅在第5步及之后可用）
    /// </summary>
    public void SkipGuide()
    {
        if (currentStep < 5) return;

        // 显示确认弹窗
        TutorialUI.ShowSkipConfirm(
            onConfirm: () =>
            {
                // 发放所有剩余步骤奖励
                for (int i = currentStep; i <= 7; i++)
                {
                    var rewards = TutorialRewardTable.GetByStep(i);
                    foreach (var reward in rewards)
                    {
                        RewardManager.GiveReward(reward);
                    }
                }

                TutorialData.SaveProgress(8);
                AnalyticsManager.Track("tutorial_skip", currentStep);
                FinishGuide();
            }
        );
    }

    /// <summary>
    /// 引导完成
    /// </summary>
    private void FinishGuide()
    {
        isGuideActive = false;
        TutorialUI.HideMask();
        TutorialUI.HideSkipButton();
        TutorialUI.PlayCelebrationAnimation();

        // 解锁全部UI
        UIMaskManager.UnlockAll();

        // 激活日常任务
        TaskManager.ActivateDailyTasks();
    }
}
```

### 9.4 对话系统实现

```csharp
// DialoguePlayer.cs
public class DialoguePlayer : MonoBehaviour
{
    [SerializeField] private TextMeshProUGUI dialogueText;
    [SerializeField] private Image speakerPortrait;
    [SerializeField] private GameObject dialoguePanel;

    private Queue<DialogueData> dialogueQueue = new Queue<DialogueData>();

    public void PlayDialogues(string[] dialogueIds, Action onComplete)
    {
        dialogueQueue.Clear();
        foreach (var id in dialogueIds)
        {
            dialogueQueue.Enqueue(TutorialDialogueTable.Get(id));
        }
        StartCoroutine(PlayNext(onComplete));
    }

    private IEnumerator PlayNext(Action onComplete)
    {
        while (dialogueQueue.Count > 0)
        {
            var dialogue = dialogueQueue.Dequeue();
            // 更新立绘表情
            speakerPortrait.sprite = LoadExpression(dialogue.expression);
            // 打字机效果
            yield return StartCoroutine(TypeText(dialogue.content, dialogue.duration_ms));
            // 等待剩余时间
            yield return new WaitForSeconds(0.5f);
        }
        onComplete?.Invoke();
    }

    private IEnumerator TypeText(string text, int durationMs)
    {
        dialogueText.text = "";
        float charDelay = (float)durationMs / 1000f / text.Length;
        foreach (char c in text)
        {
            dialogueText.text += c;
            yield return new WaitForSeconds(charDelay);
        }
    }
}
```

```csharp
// TutorialData.cs
public static class TutorialData
{
    private const string SAVE_KEY = "tutorial_progress";

    [Serializable]
    public class SaveData
    {
        public int currentStep;
        public int[] completedSteps;
        public long totalGuideTime;
        public int skipCount;
        public string lastLoginDate;
    }

    public static void SaveProgress(int step)
    {
        var data = LoadProgress();
        data.currentStep = step;
        var json = JsonUtility.ToJson(data);
        PlayerPrefs.SetString(SAVE_KEY, json);
        PlayerPrefs.Save();

        // 同步服务端
        NetworkManager.Send("tutorial/save", new { step = step });
    }

    public static SaveData LoadProgress()
    {
        if (PlayerPrefs.HasKey(SAVE_KEY))
        {
            var json = PlayerPrefs.GetString(SAVE_KEY);
            return JsonUtility.FromJson<SaveData>(json);
        }
        return new SaveData { currentStep = 0, completedSteps = new int[0] };
    }
}
```

### 9.7 关键注意事项

1. **引导优先级**：引导系统的事件监听优先级必须高于其他UI系统
2. **内存管理**：对话语音和动画资源使用Addressable加载，引导结束后释放
3. **异常恢复**：每步完成后立即保存进度，防止崩溃导致重复引导
4. **性能优化**：遮罩使用Material Property Block而非创建新Material
5. **适配处理**：高亮区域坐标需根据不同分辨率（iPhone SE ~ iPhone 15 Pro Max）动态计算
6. **测试覆盖**：需覆盖断网/杀进程/内存不足等异常场景的自动化测试用例
