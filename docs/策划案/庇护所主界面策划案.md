## 三、主展示区域（看板娘）

### 3.1 看板娘系统

**定义**：主界面中央展示的融合姬立绘，玩家可以选择已拥有的任意融合姬作为看板娘。

#### 功能说明

| 功能 | 说明 |
|------|------|
| 更换看板娘 | 长按看板娘或点击设置按钮，打开融合姬选择界面 |
| 互动 | 点击看板娘触发随机语音/动作（每融合姬有3-5条待机语音） |
| 着装显示 | 看板娘显示该融合姬的当前立绘（品质卡框效果） |
| 默认看板娘 | 未选择时，默认显示万根之母·秦若兰（LE-001） |

#### 看板娘选择界面

- 显示所有已拥有的融合姬列表
- 按品质/种族/元素筛选
- 选中后立即替换主界面看板娘
- 不影响战斗队伍配置

### 3.2 活动入口

**位置**：主展示区域右上角

**设计**：
- 常驻入口按钮，始终显示
- 有进行中的活动时，按钮高亮+红点提示+活动倒计时
- 无活动时，按钮置灰显示"暂无活动"
- 点击进入活动列表界面

#### 活动列表界面

| 内容 | 说明 |
|------|------|
| 当前活动 | 进行中的限时活动列表，显示名称、剩余时间、奖励预览 |
| 活动详情 | 点击进入活动专属页面 |
| 历史活动 | 已结束的活动（仅展示，不可参与） |

**定位**：纯展示，无属性加成

#### 头像框获取方式

| 来源 | 示例 |
|------|------|

| 品质 | 视觉效果 | 稀有度 |
|------|---------|--------|
| 普通 | 简单金属边框 | 常见 |
| 精良 | 带有元素色光效 | 少见 |
| 史诗 | 动态光效+粒子 | 稀有 |
| 传说 | 全息投影效果+专属动画 | 极稀有 |

#### 5.2.2 队伍

点击后进入队伍管理界面，功能包括：

| 功能 | 说明 |
|------|------|
| 编辑卡组 | 选择3张融合姬+1张强化芯片组成战斗队伍 |
| 融合姬列表 | 查看所有已拥有的融合姬 |

| 功能 | 说明 |
|------|------|
| 野外地图 | 10张野外地图，线性关卡推进 |
| 禁区入口 | 20个禁区，分支选路探索 |
| 关卡详情 | 显示敌人配置、奖励预览、准入条件 |
| 扫荡 | 已通关的关卡可快速扫荡（获得经验+货币，无首通奖励） |

**新增界面约束：**

- 探索页不再切成“整屏独立界面”覆盖全部主框架
- 进入野外地图/禁区列表时，必须保留：
  - 顶部状态栏
  - 底部导航栏
- 野外与禁区列表应作为主界面中部内容区的子视图进行滚动
- 地图列表内容不得侵占顶部状态栏与底部导航栏的可视区域

#### 5.2.4 任务

点击后进入任务界面，包含两个子标签：

| 子标签 | 内容 |
|--------|------|
| 每日任务 | 每日刷新的任务列表，完成后领取奖励 |
| 成就 | 永久性成就列表，达成后领取奖励 |

#### 5.2.5 商店

点击后进入商店界面，包含多个子标签：

| 子标签 | 内容 |
|--------|------|
| 源核商店 | 用源核购买高级道具 |
| 菌丝商店 | 用菌丝购买基础物资 |
| 星币商店 | 用星币购买付费内容 |
| 碎片商店 | 用碎片兑换指定角色 |
| 抽卡 | 源核抽卡招募融合姬（常驻池/限定池） |

#### 5.2.6 系统

点击后弹出系统菜单面板：

| 功能 | 说明 |
|------|------|
| 绑定账号 | 预留入口，当前版本未接入任何SDK |
| 重置账号 | 清空当前账号下的本地游戏进度并重新开始，需二次确认 |

**说明：**

- 当前版本不提供“切换账号”入口
- 当前版本不在系统菜单中承载图鉴、公告、客服、关于等次级入口
- 系统菜单当前仅聚焦账号相关操作
- 绑定账号功能先预留文案与按钮状态，待SDK接入后再补完整流程

#### 账号管理交互补充

**绑定账号**

- 当前状态：预留
- 点击后提示：`账号绑定功能暂未接入SDK`
- 不改变任何本地存档数据

**重置账号**

- 用于清空当前账号下的全部游戏内数据，重新开始新进度
- 点击后必须二次确认：
  - 第一次确认：提示“将清空当前账号进度”
  - 第二次确认：提示“该操作不可恢复”
- 确认后执行：
  - 清空当前游戏存档
  - 重建初始数据
  - 返回新账号初始状态

## 六、各功能界面交互流程

### 6.1 队伍编辑流程

```
队伍界面 → 点击"编辑队伍" → 显示已编组队伍（3融合姬+1芯片）
  → 点击空位/已有角色 → 打开融合姬/芯片选择列表
    → 选择角色 → 确认替换 → 返回队伍界面
  → 点击"保存" → 队伍配置更新
  → 点击"推荐编队" → AI根据关卡自动推荐最优队伍
```

### 6.2 探索流程

```
探索界面 → 选择地图 → 显示关卡列表（线性节点）
  → 点击关卡 → 显示关卡详情（敌人、奖励、准入条件）
    → 满足条件 → 点击"出发" → 进入战斗
    → 不满足条件 → 显示缺少条件提示
  → 点击禁区入口 → 显示禁区信息（层数、消耗抗辐射药剂）
    → 确认进入 → 消耗药剂 → 进入禁区第一层
```

### 6.3 抽卡流程

```
商店界面 → 点击"抽卡"标签 → 选择卡池（常驻/限定）
  → 显示卡池内容、概率、保底进度
  → 选择单抽(200源核)或十连(2000源核)
  → 确认消耗 → 播放抽卡动画 → 显示结果
  → 点击"再次抽取"或"返回"
```

## 七、新手引导流程

### 7.1 首次进入游戏

```
开场剧情（前任管理者的日志）→ 创建角色（输入昵称、选择头像）
→ 庇护所主界面（强制引导）
→ 引导1：点击"队伍" → 赠送初始融合姬（N品质×3）+ 初始芯片（N品质×1）
→ 引导2：编辑队伍 → 将3个融合姬和1个芯片编入队伍
→ 引导3：点击"探索" → 进入第一章废弃城区
→ 引导4：完成第1关战斗
→ 引导5：返回基地 → 点击"商店"查看商店
→ 引导6：点击"任务"查看任务系统
→ 引导7：与看板娘互动 → 引导结束
→ 自由探索
```

### 7.2 初始赠送内容

| 内容 | 数量 | 说明 |

| N品质融合姬 | 3张 | 随机N卡，确保坦克/输出/辅助各1张 |
| N品质强化芯片 | 1张 | 随机N芯片 |
| 菌丝 | 60000 | 初始基础货币 |
| 源核 | 60000 | 初始稀有代币 |
| 星币 | 60000 | 初始付费代币（测试/当前版本口径） |
| 抗辐射药剂 | 3个 | 初始禁区入场道具 |

## 八、与配置表的关联

### 8.1 相关配置表

| 配置表 | 关键字段 | 说明 |
|--------|---------|------|
| items.xlsx | 新增头像框和称号道具 | itemType为"portrait_frame"和"title" |
| achievement.xlsx | 头像框/称号奖励关联 | achievementReward中可包含头像框/称号 |
| levelUp.xlsx | 等级奖励关联 | 特定等级解锁头像框/称号 |

### 8.2 数值策划配表要点

1. 头像框和称号在items表中配置，作为特殊道具类型
2. 称号和头像框的获取条件在对应系统中配置（成就/等级/活动）
3. 看板娘不需要额外配置表，直接引用已拥有的融合姬数据
4. 新手引导赠送的初始融合姬和芯片需要单独配置

## 📋 数值策划执行指南

> 本节提取本文档中与数值配表相关的所有内容，以可直接执行的方式呈现。

### 1. 新手赠送内容配表

在 `newPlayerReward.xlsx`（或 `levelUp.xlsx` 中新增新手奖励行）中配置以下内容：

| 配置项 | 数值 | 配置字段说明 |
|--------|------|-------------|
| N品质融合姬 | 3张 | 确保坦克/输出/辅助各1张，在 `minionCards.xlsx` 中筛选 `quality=N` 且 `profession` 分别为 `tank`/`dps`/`support` 的卡，随机各选1张 |
| N品质强化芯片 | 1张 | 在 `chipCards.xlsx` 中筛选 `quality=N` 的芯片，随机选1张 |
| 菌丝（mycelium） | 60000 | 货币类型 `mycelium`，直接发放到玩家钱包 |
| 源核（sourceCore） | 60000 | 货币类型 `sourceCore`，直接发放到玩家钱包 |
| 星币（starCoin） | 60000 | 货币类型 `starCoin`，直接发放到玩家钱包 |
| 抗辐射药剂 | 3个 | 道具类型 `consumable`，道具ID在 `items.xlsx` 中配置 |

### 2. 每日免费药剂数值

| 配置项 | 数值 | 说明 |
|--------|------|------|
| 免费药剂刷新间隔 | 3600秒（1小时） | 在 `shopConfig.xlsx` 或系统常量中配置 |
| 免费药剂上限 | 5个 | 同时最多持有5个免费药剂 |
| 药剂购买价格 | 源核商店定价 | 在 `shopItems.xlsx` 中配置 |

### 3. 等级系统数值

| 配置项 | 范围 | 配置表 |
|--------|------|--------|
| 账号等级范围 | 1-99级 | `levelUp.xlsx` |
| 等级经验表 | 每级所需经验递增 | `levelUp.xlsx` 的 `expRequired` 字段 |
| 等级奖励（头像框/称号） | Lv.50、Lv.99等特定等级 | `levelUp.xlsx` 的 `rewards` 字段 |

### 4. 抽卡数值

| 配置项 | 数值 | 说明 |
|--------|------|------|
| 单抽消耗 | 200源核 | `gachaConfig.xlsx` |
| 十连消耗 | 2000源核 | `gachaConfig.xlsx` |
| 卡池类型 | 常驻池/限定池 | `gachaPool.xlsx` |

### 5. 关联配置表清单

| 配置表 | 用途 | 关键字段 |
|--------|------|---------|
| `items.xlsx` | 头像框（itemType=portrait_frame）、称号（itemType=title）、抗辐射药剂 | itemId, itemType, quality |
| `achievement.xlsx` | 成就奖励中的头像框/称号关联 | rewards, titleReward, frameReward |
| `levelUp.xlsx` | 等级经验、等级奖励 | level, expRequired, rewards |
| `minionCards.xlsx` | 新手赠送融合姬选取池 | quality, profession |
| `chipCards.xlsx` | 新手赠送芯片选取池 | quality |
| `shopItems.xlsx` | 商店道具定价、免费药剂配置 | itemId, price, currencyType |
| `gachaConfig.xlsx` | 抽卡消耗、概率 | singleCost, tenCost, rates |

### 5. 每日免费药剂逻辑

```javascript
// 免费药剂系统
const FREE_MEDICINE_CONFIG = {
  interval: 3600,    // 1小时刷新1个
  maxStock: 5,       // 上限5个
  storageKey: 'free_medicine_data',
};

function getFreeMedicineData() {
  return JSON.parse(localStorage.getItem(FREE_MEDICINE_CONFIG.storageKey) || '{}');
}

function checkFreeMedicineRefresh() {
  const data = getFreeMedicineData();
  const now = Date.now();
  
  if (!data.lastClaimTime) {
    // 首次，给1个
    data.stock = 1;
    data.lastClaimTime = now;
  } else {
    // 计算应补充的数量
    const elapsed = Math.floor((now - data.lastClaimTime) / 1000);
    const shouldAdd = Math.min(
      Math.floor(elapsed / FREE_MEDICINE_CONFIG.interval),
      FREE_MEDICINE_CONFIG.maxStock - (data.stock || 0)
    );
    if (shouldAdd > 0) {
      data.stock = (data.stock || 0) + shouldAdd;
      data.lastClaimTime = now;
    }
  }
  
  localStorage.setItem(FREE_MEDICINE_CONFIG.storageKey, JSON.stringify(data));
  return data.stock;
}

function claimFreeMedicine() {
  const data = getFreeMedicineData();
  if (data.stock > 0) {
    data.stock--;
    data.lastClaimTime = Date.now();
    localStorage.setItem(FREE_MEDICINE_CONFIG.storageKey, JSON.stringify(data));
    addItem('anti_radiation_medicine', 1);
    return true;
  }
  return false; // 无免费药剂可领
}
```
