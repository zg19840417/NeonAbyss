# 第一阶段模块拆分

> 版本：Cocos v1.0  
> 日期：2026-04-08  
> 用途：定义第一阶段 Cocos 项目的模块边界、依赖关系与实现顺序。  
> 面向：程序 AI

## 1. 第一阶段目标

第一阶段要完成的是：

- 开场剧情
- 初始 3 个融合姬发放
- 第一章 10 关主线推进
- 体力消耗与恢复
- 队伍编成
- 自动战斗
- 星级结算
- 掉落组奖励
- 融合姬升级与升品质

## 2. 第一阶段核心模块

### 2.1 启动与全局模块

目录：

- `cocos/assets/scripts/core`

职责：

- 游戏启动
- 分辨率与适配
- 全局事件
- 全局常量入口
- 场景切换入口

### 2.2 静态配置读取模块

目录：

- `cocos/assets/scripts/config`

职责：

- 读取 JSON 配置
- 提供统一查询接口
- 不保存运行时状态

第一阶段最少要有：

- `GlobalConfigRepo`
- `FusionGirlRepo`
- `PortraitSetRepo`
- `PortraitFragmentRepo`
- `StageRepo`
- `EnemyGroupRepo`
- `DropGroupRepo`
- `ItemRepo`
- `ShopRepo`

### 2.3 类型定义模块

目录：

- `cocos/assets/scripts/data`

职责：

- 定义 TypeScript 数据结构

第一阶段至少要定义：

- `FusionGirlConfig`
- `PortraitSetConfig`
- `PortraitFragmentConfig`
- `StageConfig`
- `EnemyGroupConfig`
- `DropGroupConfig`
- `ItemConfig`
- `ShopItemConfig`
- `SaveData`
- `PlayerResourceState`

### 2.4 运行时状态模块

目录：

- `cocos/assets/scripts/runtime`

职责：

- 保存玩家当前状态
- 作为 UI、剧情、战斗的唯一运行时数据来源

第一阶段至少需要：

- `PlayerState`
- `FusionGirlRuntimeState`
- `ChapterProgressState`
- `InventoryState`
- `ShopState`

### 2.5 管理器模块

目录：

- `cocos/assets/scripts/managers`

第一阶段至少需要：

- `SaveManager`
- `ResourceManager`
- `FusionGirlManager`
- `StageManager`
- `RewardManager`
- `ShopManager`
- `StoryManager`
- `BattleManager`

## 3. 页面与场景模块

### 3.1 必做场景

目录：

- `cocos/assets/scenes`
- `cocos/assets/scripts/scenes`

第一阶段必须有：

- 主菜单
- 庇护所
- 队伍
- 主线地图
- 商店
- 设置
- 战斗

### 3.2 必做 UI 组件

目录：

- `cocos/assets/prefabs`
- `cocos/assets/scripts/ui`

第一阶段建议做成独立 Prefab 的组件：

- 顶部资源栏
- 底部导航栏
- 通用按钮
- 通用弹窗
- 血条
- 页签头
- 技能图标格
- 能力图标格
- 关卡节点
- 奖励弹窗

## 4. 战斗模块

目录：

- `cocos/assets/scripts/battle`

第一阶段至少拆成：

- 战斗单位快照
- 战斗流程控制
- 行动顺序计算
- 自动技能选择
- 日志记录
- 结算输出

## 5. 剧情模块

目录：

- `cocos/assets/scripts/story`

第一阶段必须支持：

- 开场剧情脚本
- 剧情发放融合姬卡片道具
- 剧情解锁章节与关卡
- 剧情结束后切换到庇护所

## 6. 模块依赖顺序

第一阶段建议实现顺序：

1. `data`
2. `config`
3. `runtime`
4. `managers`
5. `ui`
6. `scenes`
7. `battle`
8. `story`

## 7. 第一阶段禁止事项

- 不继续引入 Phaser 风格的手写热区系统
- 不让多个 UI 层同时接收点击
- 不让配置与运行时状态混写
- 不把旧项目代码直接复制进 `cocos/`
