# 数据流与存档结构规范

> 版本：Cocos v1.0  
> 日期：2026-04-08  
> 用途：定义第一阶段 Cocos 项目的数据流与存档结构。  
> 面向：程序 AI

## 1. 总体原则

- Excel 为源头
- JSON 为配置产物
- 运行时状态独立于配置
- 存档只保存玩家状态，不保存静态配置副本

## 2. 第一阶段核心配置表

- 融合姬主表
- 立绘套装表
- 立绘碎片表
- 关卡表
- 敌方编组表
- 掉落组表
- 道具表
- 全局配置表
- 商店表

## 3. 第一阶段核心运行时数据

### 3.1 玩家资源

- 菌丝
- 源核
- 星币
- 体力
- 水点
- 火点
- 风点

### 3.2 角色状态

- 已解锁融合姬列表
- 每个融合姬当前：
  - 品质
  - 等级
  - 经验或升级进度
  - 已上阵状态
  - 立绘碎片进度
  - 已累计碎片属性
  - 待升品质资格

### 3.3 主线状态

- 当前章节
- 当前已解锁关卡
- 已通关关卡
- 已获得首通奖励记录
- 当前星级记录

### 3.4 背包与道具

- 玩家道具库存
- 道具是否自动使用、是否可见，由道具表字段决定

## 4. 道具规则

道具表第一阶段核心字段至少要支持：

- `item_id`
- `item_name`
- `item_type`
- `auto_use`
- `visible`

### 4.1 第一阶段特殊道具

- 融合姬卡片道具
  - `auto_use = true`
  - `visible = false`
  - 效果：解锁对应融合姬

## 5. 关卡奖励数据流

### 5.1 关卡表字段

至少包含：

- `stage_id`
- `chapter_id`
- `stage_name`
- `stamina_cost`
- `star_round_limit`
- `enemy_group_id`
- `first_clear_1star_drop_group`
- `first_clear_2star_drop_group`
- `first_clear_3star_drop_group`
- `repeat_drop_group`

### 5.2 掉落组规则

掉落组表至少支持：

- 固定掉落部分
- 随机掉落部分
- 二者并存

### 5.3 奖励结算顺序

1. 判定战斗胜负
2. 判定星级
3. 如果是首通，结算星级对应的一次性掉落组
4. 如果是重复挑战，结算重复掉落组
5. 发放道具
6. 若道具 `auto_use = true`，立即执行使用逻辑
7. 保存存档

## 6. 体力数据流

- 进入关卡前检查当前体力
- 若不足：
  - 弹提示
  - 禁止进入
- 若足够：
  - 扣除 `stamina_cost`
  - 进入关卡

### 6.1 体力恢复

- 恢复规则来自全局配置表
- 当前已定规则：
  - 每 5 分钟恢复 1 点

## 7. 存档结构原则

### 7.1 存档内容

存档只保存：

- 玩家资源
- 玩家角色状态
- 主线推进状态
- 道具库存
- 商店购买状态
- 设置项

### 7.2 不进入存档的内容

- Excel 配表原始数据
- JSON 静态配置副本
- 运行中临时战斗对象

## 8. 第一阶段完成标准

- 能从 JSON 读取配置
- 能创建默认运行时数据
- 能保存与读取存档
- 能完成：
  - 解锁融合姬
  - 扣体力
  - 发奖励
  - 自动使用融合姬卡片
  - 记录关卡首通与星级
