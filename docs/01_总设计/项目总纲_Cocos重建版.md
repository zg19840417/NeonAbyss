# 废土元年 Cocos 重建版项目总纲

> 版本：v1.0  
> 日期：2026-04-08  
> 用途：作为 Cocos 新项目的唯一总纲文档，供策划 AI、美术 AI、程序 AI 统一理解项目目标与边界。

## 1. 项目定位

- 类型：竖屏 2D 废土题材卡牌养成 + 自动战斗 + 主线推进小游戏
- 目标平台：微信小游戏、抖音小游戏
- 新项目引擎：Cocos Creator
- 新项目语言：TypeScript
- 设计基准分辨率：`1080 x 1920`

## 2. 当前唯一可信主轴

- 主角色体系：固定融合姬
- 抽卡内容：立绘碎片，不抽整卡
- 成长路线：升级 + 升品质
- 升星系统：删除
- 战斗芯片卡：保留，仍作为独立战斗辅助系统
- 能力槽装备物：统一命名为“模组”
- 主线推进：剧情解锁融合姬、开放对应碎片池、推进地图和战斗

## 3. 核心体验顺序

玩家进入游戏后的主体验顺序应为：

1. 进入主线
2. 获得并认识初始 3 个融合姬
3. 进入基地与队伍页
4. 推进关卡战斗
5. 获得立绘碎片与成长资源
6. 提升融合姬等级与品质
7. 解锁更多剧情、地图、融合姬与碎片池

## 4. 融合姬主规则

- 融合姬是固定角色，不再采用“重复实例随从卡”主成长模型
- 融合姬品质阶段：
  - `N / R / SR / SSR / UR / LE`
- 剧情加入的融合姬统一按 `N` 品质入队
- 一个融合姬可拥有多套立绘
- 一套立绘由 4 类碎片组成：
  - `R / SR / SSR / UR`
- 每类碎片：
  - 有独立需求数量
  - 有独立属性成长配置
  - 属性类型只允许为：生命 / 攻击 / 速度 / 全属性
- 任意集满一整套立绘后，获得 1 次可手动使用的升品质资格
- 升品质前必须展示品质预览确认页
- 升品质不额外消耗资源
- 到 `LE` 后，再完成新立绘仅用于收藏，不再继续提升品质

## 5. 抽卡与碎片主规则

- 抽卡只掉落：
  - `角色 + 立绘套装 + 碎片品质`
- 当前阶段卡池仅面向“已登场/已解锁融合姬”开放
- 多余碎片不再继续增加属性
- 多余碎片自动转化为：
  - 水点
  - 火点
  - 风点
- 元素点在对应商店兑换商品

## 6. 战斗主规则

- 战斗类型：纯自动战斗
- 我方编队：3 个融合姬 + 1 张战斗芯片卡
- 战斗卡面：上方纯立绘，下方纯信息
- 战斗中点击任意卡片，可打开详情并暂停
- 日志栏点击与暂停按钮行为一致
- 已打过的普通关卡允许重复挑战

## 7. 新项目设计原则

- 文档优先于实现
- Excel 为数值源头，JSON 仅为导出结果
- 新项目不迁移 Phaser 代码，只复用文档、资源、表结构与已验证结论
- 所有新文档默认面向 AI 执行，不写团队协作文案
- 非必要不保留旧实现兼容层
