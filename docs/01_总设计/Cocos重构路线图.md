# 废土元年 Cocos 重构路线图

> 版本：v1.0  
> 日期：2026-04-08  
> 用途：定义从 Phaser 原型转向 Cocos Creator 的重构顺序与每阶段完成标准。

## 1. 重构原则

- 新项目目录：`cocos/`
- 旧 Phaser 项目：只读参考实现
- 不做旧代码迁移，只做新项目重建
- 数据继续采用 Excel 配表 -> JSON 导出 -> 程序读取
- 先搭文档与数据骨架，再搭 Cocos 工程，再做玩法页面

## 2. 六个阶段

### 阶段 1：文档与资料重构

完成标准：

- 新文档目录结构建立完成
- 总纲文档完成
- Cocos 重构路线图完成
- 总配表手册完成
- 旧临时文档移入 `archive/临时归档`

### 阶段 2：Cocos 空工程与基础规范

完成标准：

- `cocos/` 工程建立
- TypeScript 基础结构建立
- 分辨率与适配策略落地
- 资源目录规则建立
- 数据导入链路建立
- 全局管理器与存档框架可运行

### 阶段 3：主线开局与基地骨架

完成标准：

- 主菜单可进入
- 第一章开局可进入
- 玩家获得默认 3 个初始融合姬
- 基地页基础入口可用
- 队伍页能查看融合姬详情

### 阶段 4：融合姬养成闭环

完成标准：

- 融合姬升级可用
- 融合姬升品质可用
- 立绘碎片进度可展示
- 技能/能力/模组槽位解锁可展示
- 队伍编成可保存

### 阶段 5：战斗与主线推进闭环

完成标准：

- 关卡进入战斗
- 自动战斗
- 胜负结算
- 奖励发放
- 主线推进
- 解锁新融合姬与碎片池

### 阶段 6：长期循环与外围系统

完成标准：

- 抽碎片完整可用
- 多余碎片转元素点可用
- 元素点商店可用
- 图鉴/收藏可用
- 长期循环成立

## 3. 第一阶段输出物

## 4. 第二阶段输出物

- `cocos/` 空工程
- 程序目录结构
- 数据导入脚本
- 基础 UI 与场景骨架
- 存档结构定义

## 5. 当前不做的事情

- 不继续修 Phaser 旧项目主路径
- 不在 Phaser 里继续叠新功能
- 不做“旧代码兼容迁移”
- 不先做活动、每日任务、宠物、图鉴等外围系统

## 6. 阶段推进规则

- 每阶段必须有“可验证完成标准”
- 先完成主循环，再补外围系统
- 任何新规则以新文档体系为准，不再依赖旧分散文档口径
