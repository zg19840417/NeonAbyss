# 项目现状评估

更新时间：2026-04-08

## 总体判断

当前项目已经从“多系统并行试做”进入“需要统一主干口径”的阶段。

- 策划完成度：85%
- 数值完成度：55%
- 美术完成度：65%
- 程序完成度：55%

当前最核心的判断不是“缺少功能”，而是“旧系统和新系统并存，导致很多功能看起来有，实际主流程没有真正接上”。

## 策划落地情况

### 已基本定稿

- 战斗系统的自动战斗定位、卡牌两段式结构、日志与暂停合并逻辑
- 队伍页的三层结构：3 张随从大卡 + 芯片光环条幅 + 未上阵小卡列表
- 账号模型：只保留绑定账号和重置账号
- 美术立绘标准：`1024 x 1536`、`2:3`

### 仍停留在策划层或半落地

- 主副卡融合的完整产品闭环
- 图鉴系统
- 每日任务与活动
- 新手引导
- 职业能力树

## 数值落地情况

### 已落地

- 三货币体系
- 抽卡池、敌人、卡牌、世界地图、关卡等数据表
- 基础战斗属性：HP / ATK / SPD
- 掉落表和部分奖励分发

### 当前不合理点

- 实际战斗阵容一度仍在读旧 `base.team`，而不是实例卡上阵数据
- 声望等级、关卡解锁、奖励分发使用的数据口径有分裂
- 部分系统仍然把随机生成和正式配表混用，影响后续数值验证

### 已处理

- 战斗入口改为以 `MinionCardManager` 上阵卡为主数据源
- 芯片战斗入口改为以 `ChipCardManager` 已装备芯片为主数据源
- 新开局 / 重置账号 / 预加载默认值统一到一套数据入口

## 美术落地情况

### 已落地

- 融合姬立绘资源已达到 100 张
- Boss 图 5 张
- 卡框、占位图、芯片图基础资源已存在
- 战斗大卡、队伍大卡、详情页的立绘规范已明确

### 当前不合理点

- 仍有部分旧 UI 文案乱码
- 某些页面还残留旧卡框语言，与当前“立绘优先”方向不一致
- 商店、抽卡、旧招募系统的视觉风格还没有完全统一

### 已处理

- 立绘加载与静态打包已统一
- 小卡头像改为等比缩放，不再强拉
- 详情页方向已经转向“立绘优先”

## 程序落地情况

### 已落地

- Phaser + Vite 主工程
- 主菜单、基地、战斗、禁区、野外探索、商店、队伍等基础场景
- `MinionCardManager`、`ChipCardManager`、`ReputationSystem` 等核心运行时对象
- 卡牌渲染、战斗卡渲染、详情弹窗、基础存档流程

### 当前不合理点

- `PreloadScene`、`BaseScene`、旧 `SaveSystem`、各 manager 曾同时管理默认数据和本地存档
- `BaseSystem` 里的旧招募/旧队伍逻辑仍然存在，与实例卡体系并存
- 一些数据读取仍沿用历史字段，如 `reputationSystem`
- 可见乱码仍残留在部分老系统和视图中
