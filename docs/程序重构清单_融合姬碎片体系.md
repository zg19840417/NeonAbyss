## 一、目标口径

当前程序需要从“实例随从卡 + 抽整卡 + 升星”的旧结构，迁移为“固定融合姬 + 抽立绘碎片 + 升品质”的新结构。

核心口径如下：

- 融合姬是固定角色，不再以重复实例卡为主成长核心
- 抽卡只掉落：角色 + 立绘套装 + 碎片品质
- 碎片品质仅有：R / SR / SSR / UR
- 每套立绘包含4类碎片，需求数量按表配置
- 碎片在未满需求数量前提供属性成长
- 多余碎片不再提供属性，转为水 / 火 / 风点数
- 任意完成一整套立绘，可获得1次升品质资格
- 升品质手动触发，不消耗额外资源
- 升星系统整体删除
- 战斗芯片卡系统保留
- 能力槽内装备物统一命名为“模组”

## 二、必须重做的模块

### 1. 抽卡主系统

文件：

- `src/game/systems/GachaSystem.js`
- `src/game/systems/ShopSystem.js`
- `src/scenes/views/ShopView.js`
- `assets/data/json/gachaPools.json`
- `assets/data/json/portraitFragments.json`

当前问题：

- 旧实现曾直接产出整卡，现已切换为立绘碎片结果对象
- 奖励对象还是 `characters`
- 保底与品质逻辑按“整卡”设计
- 当前抽卡结果页展示的是角色卡，不是碎片

目标改造：

- 抽卡产出改为 `fragment`
- 奖励对象结构改为“角色 + 立绘套装 + 碎片品质 + 碎片类型 + 数量”
- 保底改为 SR/SSR/UR 碎片保底
- 商店抽卡结果页改为碎片结果页
- 首次十连必出 SR 碎片

### 2. 融合姬主数据模型

文件：

- `src/game/data/GameData.js`

当前问题：

- 需要确保主数据彻底以固定融合姬对象为核心
- 需要确保角色库存不再回退到旧实例卡语义
- 没有“立绘套装进度 / 碎片进度 / 升品质资格 / 已提供属性”的模型

目标改造：

- 新建融合姬主数据结构
- 至少包含：
  - 角色ID
  - 当前品质
  - 当前等级
  - 已解锁立绘列表
  - 立绘套装碎片进度
  - 可升品质状态
  - 碎片属性累计
  - 模组槽位状态
  - 宠物栏状态
- 移除 `star` 对主成长的作用

### 3. 升星/合卡系统删除

文件：

- `src/scenes/views/TeamView.js`

当前问题：

- 还保留 `starUpgrade()`
- 还保留 `mergeUpgrade()`
- UI 里仍有“升星”
- 部分属性增长仍绑在升星逻辑上

目标改造：

- 删除升星逻辑
- 删除合卡升星逻辑
- 删除相关按钮和文案
- 把成长入口切换为“升级 / 升品质 / 模组 / 宠物”

### 4. 队伍页与角色详情页

文件：

- `src/scenes/views/TeamView.js`
- `src/game/utils/CardRenderer.js`

当前问题：

- 上阵对象仍假设为实例随从卡
- 详情页仍按“卡牌实例 + 升星”展示
- 没有碎片进度区
- 没有升品质预览入口

目标改造：

- 队伍页改为“固定融合姬编队”
- 详情页增加：
  - 当前品质
  - 碎片进度总览
  - 每种碎片已提供属性
  - 可升品质提示
  - 升品质预览按钮
- 保留战斗芯片卡展示区

### 5. 奖励分发系统

文件：

- `src/game/systems/RewardManager.js`

当前问题：

- 旧奖励路径曾直接发整卡
- 奖励逻辑默认发角色实例

目标改造：

- 奖励类型增加：
  - `fusionGirlUnlock`
  - `portraitFragment`
  - `elementPoint`
- 删除旧整卡奖励路径，仅保留 `unlockFusionGirl` / `portraitFragment` / `elementPoint`

## 四、建议的重构顺序

### 第一步：先改数据，不改UI

- 新建融合姬数据结构
- 新建立绘碎片数据结构
- 新建元素点数数据结构
- 让 `GameData` 先支持新存档格式

### 第二步：替换抽卡产出

- `GachaSystem` 改成掉碎片
- `ShopSystem` 改成发碎片奖励
- `ShopView` 改成碎片结果展示

### 第三步：替换队伍与详情页

- 队伍页不再读旧实例卡升星逻辑
- 详情页加入碎片进度、升品质入口
- 删除“升星”相关按钮

### 第四步：替换奖励与剧情解锁

- `RewardManager` 改成发碎片/解锁角色/元素点
- 主线剧情奖励改成“解锁融合姬 / 解锁碎片卡池”

### 第五步：清理旧实例卡残留

- 删除旧整卡主成长路径
- 删除旧升星/合卡路径
- 清理所有玩家可见的旧文案

## 五、当前阻塞点

这些在正式开改前已经确认，无需再次讨论：

- 剧情加入的融合姬按 `N品质` 入队
- “正式进入卡池”指立绘碎片进入卡池
- 剧情文案统一使用“加入队伍 / 解锁融合姬”
- 当前具体有哪些融合姬角色，不作为程序设计前提，后续可被剧情内容替换

## 六、实施原则

- 不再围绕旧“实例随从卡”修修补补
- 遇到旧升星、旧整卡抽取、旧奖励发卡路径，优先删除而不是兼容
- 战斗芯片卡系统与模组系统必须严格区分
- 所有新字段优先从配表读取，不把具体角色内容写死进代码
