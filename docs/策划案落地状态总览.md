## 六、风险与建议

### 🔴 高风险

1. **美术资源几乎为零**（2%落地率）
   - 影响：游戏无法对外展示，所有视觉为emoji文字
   - 建议：优先启动角色立绘和UI图标制作，已有33张ComfyUI图可作为基础

2. **核心玩法系统未实现**（抽卡/融合/任务）
   - 影响：游戏无法体验核心循环（抽卡→养成→战斗→推进）
   - 建议：按 P0 优先级依次实现 QuestSystem → GachaSystem → FusionSystem

3. **配表与代码ID体系不统一**
   - 影响：gachaItems用FM001、minionCards用MC_xxx、enemies用E_M001，三套ID无法互通
   - 建议：统一为MC_前缀体系

### 🟡 中风险

4. **Equipment残留代码未清理**
   - 影响：25处BattleSystem引用、Const.js旧配置、BaseScene死代码
   - 建议：安排一次专项清理

5. **部分配表字段为空**
   - 影响：abilityPoolId全null、levelUp消耗全相同
   - 建议：安排数值策划填充

### 🟢 低风险

6. **文档命名不统一**
   - 影响：团队协作效率
   - 建议：统一为"XX系统策划案"格式
