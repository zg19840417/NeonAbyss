# UI与图标规范

> 版本：Cocos v1.0  
> 日期：2026-04-08  
> 用途：定义新 Cocos 项目的 UI 风格、图标风格与核心界面规范。  
> 面向：美术 AI / 程序 AI

## 1. 总体 UI 方向

- 目标平台：微信小游戏、抖音小游戏
- 设计基准分辨率：`1080 x 1920`
- 风格关键词：
  - 废土
  - 科技残骸
  - 直角结构
  - 冷色界面
  - 霓虹点缀

## 2. 形状规则

- 面板：优先纯矩形
- 卡片：优先纯矩形
- 弹窗：优先纯矩形
- 按钮：允许圆角，但圆角较小

不再采用大面积圆角 UI 语言。

## 3. 顶部与底部固定结构

### 3.1 顶部资源栏

第一阶段顶部栏固定显示：

- 菌丝
- 源核
- 星币
- 体力

不显示：

- 玩家头像
- 玩家名字
- 账号等级

### 3.2 底部导航栏

第一阶段底部导航固定 5 项：

- 庇护所
- 队伍
- 主线/地图
- 商店
- 设置

## 4. 队伍与战斗详情页规范

### 4.1 共同结构

融合姬详情页与战斗中卡片详情，统一采用三页签：

- 基础
- 技能
- 能力

### 4.2 详情页上半部分

- 立绘优先
- 宽度尽量吃满可用区域
- 立绘按原始比例显示
- 下半信息区尽可能压缩，让位于立绘

### 4.3 基础页排布

- 第一行：
  - 左侧：等级 + 品质 icon + 称号/名字
  - 右侧：职业 icon + 职业汉字 + 元素 icon + 元素汉字
- 第二行：
  - 左侧生命 icon
  - 右侧红色血条
  - 血条中显示当前生命/上限生命
  - 数字需带描边
- 第三行：
  - 攻击
  - 速度

### 4.4 技能页与能力页

- 技能图标必须是正方形
- 能力图标必须是正方形
- 能力图标可比技能图标更小
- 同时只显示当前选中的一个技能/能力描述
- 不同时展开多个描述

## 5. 战斗 UI 结构

- 顶部：关卡名 + 回合数
- 中间：敌方 3 张 / 我方 3 张
- 下方：日志条
- 底部：倍速、暂停

## 6. 图标风格规则

### 6.1 职业与元素

- 不再使用字母砖块
- 采用统一 icon
- icon 旁允许跟随汉字

### 6.2 品质

- 品质用 icon 表达
- 不直接写 `N / R / SR / SSR / UR / LE` 作为主展示形式

### 6.3 攻击、速度、生命

- 优先使用 icon + 数值
- 生命必须带血条

## 7. 商店与庇护所

### 7.1 庇护所

- 作为首页信息中枢
- 包含：
  - 看板娘
  - 主线进度摘要
  - 活动入口预留

### 7.2 商店

- 第一阶段只启用基础商店
- 但 UI 预留以下页签位：
  - 基础商店
  - 元素点商店
  - 活动商店
- 抽卡入口位置可预留，但不启用

## 8. 第一阶段完成标准

- 顶部资源栏风格统一
- 底部导航风格统一
- 队伍详情与战斗详情采用同一结构
- 元素、职业、品质、战斗属性 icon 具备统一风格
