# 敌人配置系统策划案

## 1. 系统概述

### 1.1 系统定位
敌人配置系统是战斗系统的核心组成部分，定义了禁区中所有敌人的属性、技能、AI行为和掉落奖励。敌人分为普通敌人和BOSS两大类，随着楼层增加，敌人强度逐渐提升。

### 1.2 敌人分类
- **普通敌人**：随机出现在各楼层，数量较多
- **BOSS敌人**：每10层出现一个，强度高，奖励丰厚
- **精英敌人**：5的倍数楼层可能出现，介于普通和BOSS之间

## 6. 技术实现

### 6.1 系统架构
- **EnemyFactory**：敌人工厂类，根据楼层生成敌人
- **Enemy**：敌人基类，包含属性、技能、AI
- **Boss**：BOSS类，继承Enemy，添加阶段机制
- **EnemyAI**：敌人AI控制器
- **DropSystem**：掉落系统

### 6.2 敌人生成算法
```javascript
// 根据楼层生成敌人队伍
function generateEnemies(floor) {
    const enemies = [];
    const enemyCount = Math.min(3 + Math.floor(floor / 10), 5);
    
    for (let i = 0; i < enemyCount; i++) {
        // 根据楼层权重选择敌人类型
        const enemyType = selectEnemyByFloor(floor);
        
        // 计算当前楼层的属性
        const scaledStats = scaleEnemyStats(enemyType, floor);
        
        // 创建敌人实例
        enemies.push(new Enemy(scaledStats));
    }
    
    return enemies;
}
```

### 6.3 存档系统
- 已击败的BOSS记录
- 当前楼层敌人配置缓存
- 掉落历史记录（用于统计）

## 7. 平衡性考虑

### 7.1 难度曲线
- **1-10层**：新手友好，敌人较弱
- **11-20层**：难度适中，需要一定策略
- **21-30层**：难度提升，需要合理搭配职业
- **31-40层**：高难度，需要装备和等级支撑
- **41-50层**：极高难度，需要完美配合

### 7.2 反作弊机制
