# 自动战斗系统策划案

## 1. 系统概述

### 1.1 系统定位
自动战斗系统是游戏的特色核心功能，允许玩家开启自动战斗模式，让队伍自动进行战斗和禁区探索。系统根据角色职业和技能自动释放技能，配合仇恨值系统实现坦克职业的仇恨吸引机制。

### 1.2 核心机制
- **自动战斗开关**：玩家可在战斗界面开启/关闭
- **自动技能释放**：根据职业定位和技能效果自动释放
- **仇恨值系统**：敌人根据仇恨值占比选择攻击目标
- **自动探索**：在禁区界面开启后自动移动和战斗

## 2. 仇恨值系统

### 2.1 仇恨值机制

| 参数 | 说明 |
|------|------|
| **基础仇恨** | 每造成1点伤害，获得1点仇恨值 |
| **坦克主定位** | 主动技能额外**+3仇恨/damage** |
| **坦克辅助定位** | 被动技能额外**+3仇恨/damage** |
| **仇恨衰减** | 每回合结束时，仇恨值降低**10%** |
| **仇恨重置** | 战斗结束后，仇恨值重置为0 |

### 2.2 仇恨值公式

```
目标被攻击概率 = 目标仇恨值 / 所有敌方目标仇恨值总和 × 100%
```

### 2.3 仇恨值计算示例

| 回合 | 铁血狂卫(坦克) | 幻影战将(输出) | 纳米裁决(治疗) | 总仇恨 | 坦克概率 |
|------|----------------|----------------|---------------|--------|----------|
| 初始 | 0 | 0 | 0 | 0 | 33% |
| 第1回合 | 攻击+500伤害 | 攻击+200伤害 | 治疗 | - | - |
| 仇恨累计 | 0+500×4=2000 | 0+200=200 | 0 | 2200 | 91% |
| 第2回合 | 攻击+500 | 攻击+200 | 攻击+100 | - | - |
| 衰减后 | 2000×0.9=1800+2000 | 200×0.9=180+200 | 0+100 | 4080 | 93% |

### 2.4 仇恨值实现

```javascript
class HatredSystem {
    // 基础仇恨倍率
    const BASE_HATRED_PER_DAMAGE = 1;
    
    // 坦克职业仇恨加成
    const TANK_MAIN_HATRED_BONUS = 3;  // 主定位
    const TANK_SUB_HATRED_BONUS = 3;   // 辅助定位
    
    // 回合衰减率
    const HATRED_DECAY_PER_ROUND = 0.1;
    
    // 计算伤害产生的仇恨
    calculateHatredFromDamage(damage, profession) {
        let hatred = damage * BASE_HATRED_PER_DAMAGE;
        
        // 坦克主定位
        if (profession.mainRole === 'tank') {
            hatred += damage * TANK_MAIN_HATRED_BONUS;
        }
        
        // 坦克辅助定位
        if (profession.subRole === 'tank') {
            hatred += damage * TANK_SUB_HATRED_BONUS;
        }
        
        return hatred;
    }
    
    // 每回合衰减
    decayHatred(currentHatred) {
        return currentHatred * (1 - HATRED_DECAY_PER_ROUND);
    }
    
    // 计算攻击概率
    calculateAttackProbability(targets) {
        const totalHatred = targets.reduce((sum, t) => sum + t.hatred, 0);
        return targets.map(t => ({
            target: t,
            probability: totalHatred > 0 ? t.hatred / totalHatred : 1/targets.length
        }));
    }
}
```

## 3. 自动战斗AI

### 3.1 技能释放优先级

| 优先级 | 技能类型 | 触发条件 |
|--------|----------|----------|
| 1 | 治疗技能 | 队友HP < 40% |
| 2 | 坦克技能 | 队伍需要保护 |
| 3 | 辅助buff | 队伍需要增益 |
| 4 | 辅助debuff | 敌人需要削弱 |
| 5 | 群体攻击 | 敌人数量 >= 2 |
| 6 | 单体攻击 | 优先击杀目标 |
| 7 | 普通攻击 | 无技能可用 |

### 3.2 职业AI策略

#### 坦克职业（铁血狂卫、量子壁垒、战地修甲）
| 职业 | 主动技能 | AI策略 |
|------|----------|--------|
| 铁血狂卫 | 毁灭冲锋 | 优先攻击HP最低敌人，叠加仇恨 |
| 量子壁垒 | 能量立场 | 自身HP < 50%时释放 |
| 战地修甲 | 护甲过载 | 自身HP < 60%时释放 |

#### 输出职业（幻影战将、暗影黑客、噬能猎手）
| 职业 | 主动技能 | AI策略 |
|------|----------|--------|
| 幻影战将 | 幻影突袭 | 优先击杀目标，击杀后追击 |
| 暗影黑客 | 系统入侵 | 优先攻击防御最高敌人 |
| 噬能猎手 | 能量抽取 | 自身HP < 50%时优先使用 |

#### 辅助职业（守护幽灵、病毒使者、神经联络）
| 职业 | 主动技能 | AI策略 |
|------|----------|--------|
| 守护幽灵 | 灵魂锁链 | 队友HP < 70%时释放团队减伤 |
| 病毒使者 | 病毒扩散 | 敌人数量 >= 2时优先使用 |
| 神经联络 | 意识增幅 | 每3回合使用一次全队暴击buff |

#### 治疗职业（急救先锋、纳米裁决、意识复苏）
| 职业 | 主动技能 | AI策略 |
|------|----------|--------|
| 急救先锋 | 急救脉冲 | 治疗HP最低队友，阈值40% |
| 纳米裁决 | 纳米修复 | 全体治疗，阈值60% |
| 意识复苏 | 净化脉冲 | 队友有debuff时优先使用 |

### 3.3 目标选择策略

#### 己方目标选择
| 情况 | 选择策略 |
|------|----------|
| 治疗目标 | 选择HP百分比最低的队友 |
| 护盾目标 | 选择HP百分比最低的队友 |
| buff目标 | 全体或HP最低队友 |

#### 敌方目标选择
| 情况 | 选择策略 |
|------|----------|
| 单体攻击 | 优先攻击HP最低敌人（击杀追击） |
| 群体攻击 | 无需选择，对所有敌人生效 |
| 坦克仇恨 | 高仇恨值目标（吸引火力） |

## 4. 仇恨值在战斗中的应用

### 4.1 敌人攻击逻辑

```javascript
class EnemyAI {
    // 敌人选择攻击目标
    selectTarget(allies) {
        // 计算每个目标的仇恨占比
        const probabilities = HatredSystem.calculateAttackProbability(allies);
        
        // 随机选择（根据仇恨权重）
        const random = Math.random();
        let cumulative = 0;
        
        for (const p of probabilities) {
            cumulative += p.probability;
            if (random <= cumulative) {
                return p.target;
            }
        }
        
        // 兜底：返回仇恨最高的
        return allies.reduce((max, a) => a.hatred > max.hatred ? a : max);
    }
}
```

### 4.2 战斗流程

1. **回合开始**
   - 所有人先手检测（速度判定）
   - 坦克被动技能触发（仇恨相关）
   - 护盾/buff持续时间检查

2. **行动阶段**
   - 按速度排序依次行动
   - 每个角色行动：
     - 计算仇恨值
     - 释放技能/普攻
     - 累计伤害和仇恨

3. **回合结束**
   - 仇恨值衰减（×0.9）
   - buff/debuff持续时间-1
   - 病毒等持续伤害结算

4. **战斗结束**
   - 仇恨值重置
   - 经验值结算

## 6. 界面设计

### 6.1 设计风格定位

**风格名称**：废土与神圣融合风格（Fusion Wasteland）

**设计理念**：
- 融合核战废土的沧桑感与神圣融合的神圣感
- 既有废墟的厚重感，又有生命之光的柔和
- 写实但不冰冷，神圣但不刺眼

### 6.2 战斗界面布局规范

#### 6.2.1 核心设计原则
- **横向布局**：敌我双方均采用横向排列，消除传统RPG"前后排"站位暗示
- **信息优先**：界面设计以放大数字、文本、卡片为核心，确保信息清晰易读
- **统一卡片**：敌我双方使用相同尺寸的卡片设计，通过颜色区分阵营
- **废土质感**：卡片和按钮有厚重感，边框柔和但清晰

#### 6.2.2 布局结构
```
┌─────────────────────────────────┐
│  FLOOR 15           回合 3      │  ← 顶部状态栏
├─────────────────────────────────┤
│  [敌人1] [敌人2] [敌人3]         │  ← 敌方区域（横向，最多3人）
├─────────────────────────────────┤
│         BATTLE                  │  ← 战斗播报区
│    战斗日志文字                  │
├─────────────────────────────────┤

├─────────────────────────────────┤

```
